# 文物影响评估项目资料识别规则 v0.1

本文件用于把用户给出的项目资料文件夹识别为文评报告可用的输入材料。默认先运行 `scripts/scan_project_folder.py` 生成初筛结果，再由 AI 按本规则复核。

## 识别原则

- 项目资料夹中的文件都是输入资料；不要把其中的既有成果直接视为最终成稿模板。
- `references/source-originals/` 才是成稿结构和表达参考。
- 文件名、上级目录、扩展名和正文预览共同判断资料类型；不能只看文件名。
- 图片、图纸、KML、坐标表等不能被忽略，它们往往支撑第四章空间关系和第五章影响判断。
- 合同、商务文件通常不进入正文，但可辅助确认项目名称、委托事项和报告范围。

## 资料类型

| 类型 | 常见文件名或目录线索 | 主要用途 |
| --- | --- | --- |
| 项目基础与规划 | 项目资料、规划平面图、总平面图、项目说明、可研、设计、建设方案、运营方案 | 第三章项目规划概况；第五章设计合规性、运营期影响 |
| 文物调查与审查 | 文物调查报告、文物部门函件、备案告知书、审查意见、核查报告、调查的报告 | 第一章必要性、第二章文物概况、第四章空间关系、结论依据 |
| 空间范围与坐标 | 红线、用地范围、四至坐标、坐标表、KML、OVKML、范围示意图、正射影像图 | 第四章空间分布关系；距离、重叠、管控区判断 |
| 考古调查勘探成果 | 考古调查、勘探工作报告、验收意见书、勘探单元、探孔、标准孔、剖线 | 第二章文物现状、第四章勘探工作变量、第五章地下文物影响 |
| 现场照片与图像 | 地块现状照、实地踏查照、信息采集照、正射影像、剖面图 | 第二章现状、第四章位置关系、第五章景观风貌与施工影响 |
| 成果与评审材料 | 成果资料、复核组件、评审意见表、已有文评报告 PDF | 可作为项目输入资料和审核意见来源；不得替代 `source-originals` 的成稿参考角色 |
| 商务合同 | 合同、委托书、任务书 | 辅助确认委托范围、项目名称和服务内容；一般不写入正文事实 |
| 其他 | 日志、plot.log、Thumbs.db、系统文件 | 默认不入正文，必要时作为制图过程材料留档 |

## 觉海寺输入样本剖面

`/Users/drevan01/Desktop/觉海寺项目/` 体现了典型输入结构：

| 目录 | 判断 | 对文评的主要价值 |
| --- | --- | --- |
| `1.项目资料/` | 项目基础、规划图、文物调查函件、正射影像、KML/OVKML | 项目背景、规划概况、文物关系、空间位置 |
| `2.商务资料/` | 合同和委托范围 | 辅助确认项目名称、服务内容，不作为技术结论依据 |
| `3.执行资料/1.外业成果/` | 现场照片、踏查、探孔、剖线、汇总 KML | 文物现状、地块现状、勘探过程和现场支撑 |
| `3.执行资料/3.制图成果/` | 红线、勘探区域、坐标、探孔、剖线图 | 空间关系、坐标和地下文物影响依据；勘探图不插入第四章第三节 |
| `3.执行资料/4.报告/` | 开工请示、考古调查勘探报告、验收意见 | 支持性文件、勘探结论、地下文物风险判断 |
| `4.成果资料/` | 复核组件、评审意见、最终 PDF | 作为输入样本中的成果和评审材料；写新项目时应区分“可引用事实”和“成稿参考” |

## 使用规则

- 先建立“文件名 - 类型 - 可提取事实 - 对应章节 - 风险/缺项”的资料目录。
- 对图纸、坐标表、KML 等空间资料，必须列入第四章证据候选。
- 对调查勘探报告、验收意见，必须列入第二章、第四章和第五章证据候选；第四章第三节仅用于替换固定文本变量，不插入图件。
- 对已有最终报告或评审意见，必须标记为“成果/评审材料”，防止直接照抄为新报告正文。
