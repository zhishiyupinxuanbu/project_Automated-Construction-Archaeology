# 影响评估成稿固定内容的两层模板模型

## 一、修正后的判断口径

两份成稿的固定内容不能只理解为“逐字相同文字”。应分为两层：

1. **硬模板原文**：两份报告中一模一样的标题或段落，以后一字不差保留；只在明确占位处替换项目名、文物名等变量。
2. **同类变量内容**：两份报告文字不一样，但在同一章、同一节、同一功能位置出现，说明这一类内容必须写，只是根据项目事实替换。比如地区概况、文物概述、项目必要性、选址分析、建设运营方案、影响路径、减缓措施等。

因此，后续生成报告时不能只找共同句子，也不能只保留共同标题；必须按“固定标题层级 + 固定原文段落 + 同类变量段落”三者共同拼装。

## 二、硬模板原文的范围

已经抽取到两份成稿逐字相同的标题/正文候选共 89 个，其中标题 50 个、正文段落 39 个。典型硬模板包括：

- `一、总则`
- `（一）编制背景`
- `1.评估项目基础信息`
- `2.文物影响评估必要性`
- `（二）评估目标`
- `（三）评估内容`
- `（四）评估原则`
- `（五）评估依据`
- `1.宪章公约`
- `2.法律法规`
- `3.文件规定`
- `4.技术规范`
- `5.其他资料`
- `（六）评估方法`
- `1.调查法`
- `①资料收集调查`
- `②现场调查`
- `2.综合分析法`
- `①清单对比法`
- `②矩阵法`
- `③叠图法`
- `3.预测法`
- `五、建设项目可能对文物造成的影响分析与评估`
- `六、减缓措施建议`
- `七、文物影响评估结论及建议`
- `八、支撑法律法规及文件`

硬模板原文还包括若干法规条文、评估方法解释、第五章引言、价值评估说明等完整段落。详见：`两份成稿逐字相同标题与段落.md`。

人工确认固定条目：第一章“（五）评估依据 / 1.宪章公约”六项内容和逐条编号格式固定，不得合并或改写。其中 `《会安草案——亚洲最佳保护范例》（2005）（Hoi An Protocols for Best Conservation of Historic Towns and Urban Areas）；` 按人工确认截图固定；两份成稿存在的 `一一` 字符差异不再作为后续模板写法。

人工确认固定条目：第一章“（五）评估依据 / 2.法律法规”前三条法规固定为《中华人民共和国文物保护法》《中华人民共和国文物保护法实施条例》《内蒙古自治区文物保护条例》，必须逐条编号并写明修订/施行时间。其他法规、保护条例属于项目适配补充，必须由所涉及文保点或项目资料触发，且同样写明修订或施行时间；涉及长城类文保点时可补入 `《长城保护条例》2006年12月1日起施行。`

人工确认固定条目：第一章“（五）评估依据 / 3.文件规定”七项文件规定内容及版式固定，编号后不加空格，不得改为“政策文件”、不得合并或删改文号、日期、标点。

注意：`4.技术规范`只固定标题层级，不固定其下具体规范条目。该部分通常完整摘自本项目可研报告、工程方案、设计说明、施工方案、建筑/消防/给排水/环保/运营相关工程资料中的对应章节。不得从项目资料中随意改写、概述或选择性摘抄；两份成稿中技术规范条目即使出现相同或相似，也不升格为所有文评通用模板内容。

注意：`5.其他资料`标题层级和条目写法可以固定，前两条固定为《世界文化遗产影响评估指南》（2011年）和《中国文物古迹保护准则》（2015年）；其后是否列入某项资料必须由本项目资料判断。常见写法如 `《【项目名称】可行性研究报告》；`、`《【项目名称】考古调查、勘探工作报告》；` 可作为变量模板，但只有实际资料包中存在或可确认存在时才写入；不得凭样本成稿自动补入不存在的资料，不得列入图纸、图片、正射影像、红线、KML/OVKML、坐标表、现场照片、验收意见书。

注意：`（六）评估方法`不是可自由概述的同类变量内容，而是固定正文模板。仅允许替换【项目名称】和【文物名称】，不得删减资料收集调查、现场调查、清单对比法、矩阵法、叠图法、预测法各段，也不得把 `项目信息。`、`访谈调查。`、`实地勘察。` 改成逗号连接句。

## 三、同类变量内容的范围

这些内容不一定逐字相同，但两份成稿都在同一位置展开，说明以后所有文评都不能省略，只能换成本项目事实。

第一章“评估项目基础信息”中的影响评估引入段属于固定正文段，不按普通同类变量内容自由改写。固定写法为：

```text
现结合项目当前地理位置、施工方案，以及相关法律法规和行业标准，对【文物名称】可能受到的影响进行评估。具体来看，【项目名称】在设计、建设、运营过程中，施工方式、车辆、人员等因素可能对不可移动文物本体及周边环境造成影响，为统筹推进项目实施与文物保护，实现项目建设与文化遗产保护协同发展，需系统性评估【项目名称】在建设及运营期间对所涉及文物的影响。
```

该段只替换【项目名称】和【文物名称】。

第一章“2.文物影响评估必要性”为固定正文，不按普通同类变量内容自由改写，也不替换任何项目变量。固定内容包括文物影响评估定义段、《文物保护法》第二十八条、第二十九条、内政办发〔2024〕40号和内文物发〔2025〕6号依据段。

第一章“（三）评估内容”中的收束句为固定文本，只替换项目名称：

```text
根据文物影响的程度，明确【项目名称】是否可行，提出调整建议和针对性的减缓措施。并根据【项目名称】的文物影响评估结论，提出其他相关建议。
```

不得改写、扩写或加入最终影响等级、建设可接受性、审批结论审核主体等说明。

### 第二章：地区描述

- 固定功能位置：文物所在区域概况
- 应有内容类型：地理位置；自然概况/自然环境；社会经济情况/区域发展；历史沿革/历史背景；交通或区位条件
- 生成要求：写项目所在行政区、自然地理、历史文化和交通区位，不得压成一句“项目位于某地”。

### 第二章：文物描述

- 固定功能位置：`（二）【文物名称】概述`
- 应有内容类型：文物基本情况；保存现状；调查、发掘、保护工程情况；保护范围及建设控制地带；价值陈述；价值评估
- 生成要求：按文物对象事实展开，不得只列文物名称、级别和距离。

### 第三章：项目描述

- 固定功能位置：项目概况
- 应有内容类型：项目名称、代码、单位、位置；建设目标；建设内容；建设规模
- 生成要求：项目事实必须成段叙述，并与表格互相支撑。

### 第三章：项目建设必要性

- 固定功能位置：项目建设必要性
- 应有内容类型：多条必要性理由
- 生成要求：两份报告理由不同，但都不是一句话；必须结合项目类型展开多条理由。

### 第三章：建设及运营方案

- 固定功能位置：建设及运营方案
- 应有内容类型：建设规模与技术指标；运营方案与规模/运营管理
- 生成要求：必须写工程规模、具体建设内容、运营组织或管理方式。

### 第一章：技术规范

- 固定功能位置：一、总则 / （五）评估依据 / 4.技术规范
- 应有内容类型：与本项目工程内容直接相关的技术规范、设计规范、施工规范、消防规范、环保规范、运营管理规范等
- 生成要求：标题层级固定，但条目内容不固化；通常必须从本项目可研报告、工程方案、设计说明、施工方案及相关工程资料中的对应章节完整摘录，不得机械沿用觉海寺或正蓝旗的技术规范清单，也不得改写、概述或选择性摘抄。

### 第一章：其他资料

- 固定功能位置：一、总则 / （五）评估依据 / 5.其他资料
- 应有内容类型：世界文化遗产影响评估指南、中国文物古迹保护准则、保护规划、可研报告、调查勘探报告、项目方案和其他规划文件
- 生成要求：标题层级和条目书写方式固定；前两条固定写入，后续具体条目必须逐项核对项目资料是否存在。可研报告、考古调查勘探报告等项目资料采用 `《【项目名称】【资料类型】》；` 写法，但不得在缺少资料依据时自动补写；图纸、图片、正射影像、红线、KML/OVKML、坐标表、现场照片、验收意见书不得列入本小节。

### 第四章：空间关系

- 固定功能位置：项目用地与文物空间分布关系
- 应有内容类型：相对位置；坐标/范围/距离；保护区划或缓冲区关系；图件叠合说明
- 生成要求：必须用坐标、红线、距离、区划图件支撑，不能只说“位于建设控制地带”。

### 第四章：选址分析

- 固定功能位置：`（二）【项目名称】选址分析`
- 应有内容类型：可研报告、可行性研究报告或建设方案中 `选址分析`、`选址必要性分析` 等对应章节的完整原文。
- 生成要求：有原文时完整摘取，不修改、不缩减、不概括；缺对应原文章节时待补，不得依据项目位置、文物空间关系、文物保护约束或 AI 初步分析替代。

### 第四章：勘探工作

- 固定功能位置：`（三）勘探工作`
- 应有内容类型：固定两段正文，仅替换项目名称、勘探公司和勘探结论；固定句中的实际完成面积按勘探报告原文填入。
- 生成要求：不得自行发挥，不摘写勘探方法、探孔布设、GPS 测点、验收过程等细节，不插入任何图片或图件。

### 第五章：影响分析

- 固定功能位置：第五章标题下、`（一）项目方案设计合规性评估` 前的总起段；建设期/运营期影响评估。
- 应有内容类型：固定两段总起段，仅替换项目名称和文物名称；环境影响；地下文物影响；景观风貌影响；小结；表格。
- 生成要求：总起段不得删除、改写、缩减、移动到小节内，不得替换为影响识别矩阵或 AI 概述句；第五章必须正文分析加表格，不得只给表格或结论。

### 第六章：减缓措施

- 固定功能位置：减缓措施建议
- 应有内容类型：对应第五章每条影响路径
- 生成要求：措施标题可随项目变化，但必须逐条回应影响问题，写到可执行层面。

### 第七章：结论建议

- 固定功能位置：评估结论及其他建议
- 应有内容类型：结论；保护方案；监测；应急；审批沟通；发现文物程序
- 生成要求：第七章整体为固定内容，仅作变量替换，不自由发挥。第一段按项目方案合规状态二选一：全部合规时用“基于上述分析……项目设计符合法律法规和相关规范，具有可行性。”；须调整后方可合规时用“但须……调整后……”固定句。其他建议是固定功能，不是可删尾巴。

### 第八章：法规支撑

- 固定功能位置：支撑法律法规及文件
- 应有内容类型：固定六项法规政策及条文全文
- 生成要求：第八章内容及格式完全固定，禁止删减、摘要、改写或追加专项法规。除非相关法律法规、政策文件更新版本，否则必须逐字写入 `references/11-固定正文结构与固定内容.md` 的第八章固定内容。

## 四、对 0630 干净版问题的重新表述

它的问题不是“没有完全复制觉海寺成稿”，而是同时违反了两层规则：

- 对硬模板原文：一二三四级标题、第一章依据/方法层级、第五章固定引言、法规条文等没有一字不差保留。
- 对同类变量内容：地区描述、文物描述、项目必要性、选址分析、建设运营方案、影响路径和措施展开都被压成摘要。

所以正确的后续规则应是：相同文字原样保留；不同文字按同类内容、同等层级、同等展开密度替换，不能因为项目不同就缩成短段。
