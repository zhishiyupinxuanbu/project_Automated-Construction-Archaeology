# 有分区、有遗迹模板匹配检查

检查对象：

`智能报告生成基准模板_北京卓凡文博技术有限公司_有分区、有遗迹.docx`

## 检查结论

该模板与现有“有遗迹”模板相比，普通字段占位符、遗迹表、标准孔表、剖面探孔表等结构基本一致，现有生成脚本可以替换和填入。

当前新版模板的新增差异主要是：

- 新增图片槽：`{{图:项目勘探分区示意图}}`
- 新增分区文本占位符：`{{勘探分区原因}}`、`{{勘探分区逐项说明}}`
- 空白表新增字段：`是否存在勘探分区`、`勘探分区数量`

## 已补规则

生成脚本已增加以下规则：

- `{{图:项目勘探分区示意图}}`
- 默认从制图成果目录中匹配文件名含 `勘探分区` 的图片，例如 `项目勘探分区示意图.jpg`。
- `{{勘探分区原因}}` 和 `{{勘探分区逐项说明}}` 优先从填报表读取；填报表为空时，从外业成果根目录的已填写汇总文档读取，例如 `项目现场记录-益民.docx`。
- 剖线、标准孔、遗迹详情继续从各勘探区域 `汇总/项目现场记录.docx` 读取，避免根目录汇总文档没有逐条详情时造成漏填。
- 若空白表“标准孔”工作表没有 `探孔编号` 列，生成时会重新从制图成果 `标准孔坐标.xlsx` 补回 TK 编号，以匹配现场记录中的标准孔描述。

## 仍需注意

空白表目前只有 `是否存在勘探分区`、`勘探分区数量` 两个分区字段。只要项目资料中有已填写汇总文档，生成器可以直接从汇总文档读取分区原因和逐项说明。

如果后续某个项目没有已填写汇总文档，建议在空白表中补充：

- `勘探分区原因`
- `勘探分区逐项说明`

若采用循环块方案，建议新增工作表 `勘探分区`，表头为：

- `分区名称`
- `分区位置`
- `地形地貌`
- `土质植被`
- `勘探条件`
- `涉及文物或重点区域`
- `普探孔距`
- `重点孔距`
- `分区描述`

其中 `分区描述` 可人工直接填写；如果为空，后续脚本可用前面字段组合生成。

## 模板选择规则建议

模板选择已按以下规则处理：

- `无遗迹`
- `有遗迹`
- `有分区、有遗迹`
- 可选：`有分区、无遗迹`

判断依据：

- `遗迹结论` 判断 `有遗迹/无遗迹`。
- `是否划分勘探分区=是` 或 `勘探分区数量>0` 判断 `有分区`。
- 同时满足有分区和有遗迹时，优先选择文件名包含 `有分区、有遗迹` 的基准模板。

## 文件命名规则

有分区模板建议继续使用：

```text
智能报告生成基准模板_公司全称_有分区、有遗迹.docx
```

如以后增加无遗迹分区模板，建议命名：

```text
智能报告生成基准模板_公司全称_有分区、无遗迹.docx
```
