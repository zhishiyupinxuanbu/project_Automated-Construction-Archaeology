# 智能勘探报告通用流程

## 目标

智能勘探报告项目应形成一个可复用的生产流程：输入任意符合资料目录规范的项目资料包，系统自动完成资料识别、人工填写表生成、模板推荐、正式报告生成和专项检查，最终输出完整、专业、可审计的考古调查、勘探工作报告。

这个流程面向多家公司、多项目、多资料形态，不绑定某一个示例项目。

## 流程总览

```text
项目资料目录
  ↓ prepare
人工填写表（给用户确认事实）
  ↓ build-form
智能报告信息填报表（机器生成输入）
  ↓ generate
正式报告 DOCX
  ↓ check_smart_report
检查表 XLSX
```

## 阶段 0：系统自检

命令：

```bash
python3 智能生成报告技能资料/工具脚本/run_smart_report_workflow.py doctor
```

用途：

- 检查核心目录是否存在。
- 检查核心脚本是否存在。
- 检查人工填写表模板和默认智能基准模板是否存在。
- 盘点多家公司 `有遗迹/无遗迹` 的智能基准模板矩阵和旧模板升级来源。
- 检查 `.env.local` 是否配置区域概况 API key（`ARK_API_KEY` / `DOUBAO_ARK_API_KEY`）。

验收标准：

- 智能基准模板矩阵覆盖当前要支持的公司和 `有遗迹/无遗迹` 状态。
- 旧模板只能作为升级来源，不作为正式报告生成模板。
- 核心脚本、模板、输出目录均存在。

如发现智能基准模板缺口，先运行：

```bash
python3 智能生成报告技能资料/工具脚本/run_smart_report_workflow.py upgrade-templates
```

该命令默认只补缺失模板，不覆盖已有基准模板；北京卓凡文博技术有限公司无遗迹基准模板按已定稿模板保护。

## 阶段 1：项目资料识别和人工填写表

命令：

```bash
python3 智能生成报告技能资料/工具脚本/run_smart_report_workflow.py prepare 基础信息/项目目录
```

输入：

- 项目资料目录。

输出：

- `过程资料/项目名称/人工填写表/项目_人工填写表.xlsx`

系统应自动识别：

- 项目名称
- 建设单位
- 勘探单位
- 项目位置
- 项目面积、调查面积、勘探面积
- 勘探时间
- 遗迹结论
- 文物概况类型
- 图纸目录
- 外业照片目录
- 既有报告目录

人工确认点：

- 用户打开人工填写表，确认自动识别内容是否准确。
- 用户补齐系统无法可靠识别的信息。

验收标准：

- 不修改 `基础信息/`。
- 不覆盖已有人工填写表，除非显式指定覆盖。
- 输出模板推荐结果，说明是否找到对应公司/遗迹状态的智能基准模板和旧模板升级来源。

## 阶段 2：智能报告信息填报表

命令：

```bash
python3 智能生成报告技能资料/工具脚本/run_smart_report_workflow.py build-form 过程资料/项目名称/人工填写表/项目_人工填写表.xlsx
```

输入：

- 已确认的人工填写表。

输出：

- `过程资料/项目名称/人工填写表/项目_智能报告信息填报表_确认生成.xlsx`

系统应做：

- 将人工填写表中的基础字段写入智能填报表。
- 保留智能填报表的多工作表结构。
- 检查必填字段是否完整。
- 再次推荐模板。

下一步增强方向：

- 自动导入红线坐标、勘探单元、标准孔、剖线坐标。
- 自动整理图片清单。
- 自动识别回函 PDF/OCR 结果。
- 自动导入现场记录 DOCX 中的剖线和标准孔描述。

验收标准：

- 智能填报表结构完整。
- 必填字段检查清晰。
- 不把公司模板固定信息当成项目字段覆盖。

## 阶段 3：模板推荐和确认

模板推荐依据：

```text
勘探单位 + 遗迹状态
```

规则：

- `勘探单位` 只用于选择模板。
- 公司名称、人员、资质附件、页眉页脚、落款等公司固定信息由模板保留。
- 项目事实由智能填报表替换。
- 若缺少某公司/遗迹状态的智能基准模板，正式生成必须停止。旧模板只能先升级为智能基准模板，不能直接参与正式生成。

命令：

```bash
python3 智能生成报告技能资料/工具脚本/run_smart_report_workflow.py recommend 过程资料/项目名称/人工填写表/项目_智能报告信息填报表_确认生成.xlsx --smart
```

验收标准：

- 能明确指出智能基准模板是否存在。
- 能明确指出旧模板升级来源是否存在。
- 如果智能基准模板缺失，要作为流程缺口记录，并先补齐后再生成。

## 阶段 4：正式报告生成

命令：

```bash
python3 智能生成报告技能资料/工具脚本/run_smart_report_workflow.py generate \
  --form 过程资料/项目名称/人工填写表/项目_智能报告信息填报表_确认生成.xlsx \
  --project-dir 基础信息/项目目录 \
  --use-region-api
```

输出：

- `生成报告/项目名称-智能生成勘探报告.docx`
- `生成报告/项目名称-智能生成勘探报告-专项检查表.xlsx`

内部生成检查 txt 默认转存到 `过程资料/项目名称/生成检查/`，只作为检查表输入；最终 `生成报告/项目名称/` 文件夹只保留 DOCX 和一个 XLSX。

正式生成报告时必须调用 `--use-region-api`，使输出 DOCX 已经包含完整 `项目区域概况`。初版生成后，后续所有修改都应先将当前报告复制到 `过程资料/项目名称/备份/` 作为回档备份，再继续修改当前报告；用户确认定稿后，`生成报告/项目名称/` 只保留定稿报告和最终检查 XLSX。

生成器应处理：

- 普通字段替换。
- 条件段落。
- 项目区域概况。
- 历史沿革长段自动分段。
- 项目建设内容为空时删除空句。
- 项目地理坐标为空时删除空坐标尾句。
- 勘探单元编号连续时略写。
- 红线坐标、勘探单元、标准孔、剖线等重复结构。
- 图纸和照片插入。
- 图号自动重排。
- TOC 目录域打开时更新。

验收标准：

- 未解析占位符为 0。
- 必填字段不为空。
- 自动插图数量和来源可检查。
- 检查 XLSX 记录关键派生字段和处理动作。

## 阶段 5：专项检查

专项检查输出：

- Excel 检查表

检查维度：

- DOCX 结构。
- 未解析占位符。
- 字段替换。
- 图片覆盖。
- 已匹配图片槽位。
- 未插入图片明细。
- 错误插入与失败原因。
- 外部资料导入证据。

质量分级目标：

- 阻断：模板选错、必填字段为空、占位符未替换、关键图件缺失。
- 警告：区域概况过短、历史沿革未分段、图片覆盖不足、目录需 Word 打开更新。
- 提示：备用图片未插入、源资料中存在未用文件。

## 多家公司支持原则

项目最终应维护一套智能基准模板矩阵：

```text
智能生成报告技能资料/知识库/3.模板与表单/
  规则/
  模板/
    报告、计划/
      智能报告生成基准模板_公司全称_无遗迹.docx
      智能报告生成基准模板_公司全称_有遗迹.docx
    表格模板/
      人工填写表模板.xlsx
```

每家公司模板都应做到：

- 占位符命名一致。
- 图片槽位规则一致。
- 重复块规则一致。
- 公司固定信息保留。
- 可通过同一套生成器和检查器处理。

## 当前优先级

1. 补齐 `build-form` 的资料自动导入能力。
2. 为内蒙古煊迹考古勘探有限公司建立智能基准模板，作为多家公司样板。
3. 把图片槽位规则表结构化。
4. 把专项检查升级为质量门禁。
5. 再逐步补齐其他公司智能基准模板。
