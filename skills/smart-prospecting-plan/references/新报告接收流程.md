# 新报告接收流程

本文档用于把 Codex 本地记忆中的“新报告 intake 工作流”迁入项目。后续同事拿到项目文件夹后，应以本文档和 `AGENTS.md` 为准，不依赖任何本机隐藏记忆。

## 适用场景

当用户要求生成新的考古调查勘探报告，且尚未明确项目源资料文件夹、填报表或模板时，必须使用本流程。

不适用于：

- 单纯清理、整理目录。
- 已经确认源资料、填报表和模板后的继续生成或问题修复。
- 与本项目结构无关的其他文档工作。

## 当前目录口径

- `基础信息/`：放准备写报告的项目源资料文件夹，只读，不写生成结果。
- `过程资料/`：只放生成过程中的可再生过程文档，例如预填表、OCR 结果、区域概况 JSON、临时检查材料和诊断留痕；删除后不应影响智能生成报告系统运行。
- `过程资料/项目名称/人工填写表/`：放给用户确认和补填的人工填写表。
- `生成报告/`：放最终报告/计划和一个检查 XLSX 表格。报告生成完成后，项目子文件夹中不保留生成检查 txt、专项检查 md、临时诊断或其他过程文件。
- `智能生成报告技能资料/`：独立核心资料包，顶层只保留 `知识库/`、`工具脚本/`、`项目说明/` 三个文件夹。
- `智能生成报告技能资料/知识库/3.模板与表单/规则/`：模板填充、字段计算和匹配规则。
- `智能生成报告技能资料/知识库/3.模板与表单/模板/报告、计划/`：当前智能报告和计划基准模板来源。
- `智能生成报告技能资料/知识库/3.模板与表单/模板/表格模板/`：当前唯一人工填写表模板来源。

关键脚本、模板和长期规则都必须归入 `智能生成报告技能资料/` 的三类目录中；不要把脚本、本机虚拟环境或系统运行依赖放在 `过程资料/`。

旧路径 `生成成果/智能生成报告技能资料/`、`生成成果/智能模板/`、`生成成果/正式报告/` 已不再作为当前项目口径。

## 需要先确认的信息

开始新报告前，先确认：

1. 当前工作目录是否为 `/Users/ivan/Desktop/智能勘探报告` 或同结构迁移副本。
2. 用户是否已经提供项目源资料文件夹。
3. 目标项目是否已经有完整 `智能报告信息填报表`。
4. 模板选择是否已经由用户确认。
5. 这是全新生成任务，还是已经确认过资料和模板的继续任务。

## 标准闸门

新报告必须经过三个确认闸门：

1. 用户确认项目源资料文件夹和源资料预检结果。
2. 用户确认人工填写表已经补全。
3. 用户确认系统推荐的模板正确。

在这些闸门完成前，不要直接生成正式报告；在第一个闸门完成前，也不要生成或覆盖人工填写表。

## 操作步骤

1. 如果项目源资料文件夹不明确，先询问用户提供路径。

   常见位置：

   ```text
   基础信息/项目目录
   ```

2. 源资料路径明确后，先做源资料预检，只检查不生成。

   ```bash
   python3 -B 智能生成报告技能资料/工具脚本/run_smart_report_workflow.py preflight "基础信息/项目目录"
   ```

   把预检结果交给用户确认。用户未确认前，不得进入 `prepare`。
   预检和自动发现目录时，内业/制图成果除标准 `3.执行资料/3.内业成果`、`3.执行资料/3.制图成果` 外，也要兼容 `3.执行资料/4.内业成果`；外业成果除 `3.执行资料/1.外业成果` 外，也要兼容 `3.执行资料/2.外业成果`；既有报告目录兼容 `4.报告` 与 `5.报告`。如自动发现仍不准确，正式生成时必须显式传入 `--drawings-dir`、`--photos-dir`。

3. 用户确认项目源资料文件夹和预检结果后，生成或定位人工填写表。

   默认使用完整 `智能报告信息填报表` 风格，不使用精简确认表。

   ```bash
   python3 -B 智能生成报告技能资料/工具脚本/run_smart_report_workflow.py prepare --source-confirmed "基础信息/项目目录"
   ```

   输出通常在：

   ```text
   过程资料/项目名称/人工填写表/
   ```

4. 把生成的人工填写表路径告诉用户，等待用户确认已经补全。

5. 用户确认表格完成后，读取表格并推荐模板。

   ```bash
   python3 -B 智能生成报告技能资料/工具脚本/run_smart_report_workflow.py recommend "过程资料/项目名称/人工填写表/项目名称_智能报告信息填报表_确认生成.xlsx" --smart
   ```

   模板选择依据：

   - `勘探单位`
   - 遗迹状态，如 `无遗迹`、`有遗迹`
   - 是否存在勘探分区
   - `智能生成报告技能资料/知识库/3.模板与表单/模板/报告、计划/` 中的智能基准模板

6. 明确告诉用户选中的模板，并等待用户确认。

7. 用户确认模板后，生成正式报告。

   ```bash
   python3 -B 智能生成报告技能资料/工具脚本/fill_smart_template_from_form.py \
     --form "过程资料/项目名称/人工填写表/项目名称_智能报告信息填报表_确认生成.xlsx" \
     --drawings-dir "基础信息/项目目录/3.执行资料/3.制图成果" \
     --photos-dir "基础信息/项目目录/3.执行资料/1.外业成果" \
     --use-region-api \
     --output "生成报告/项目名称-智能生成勘探报告.docx"
   ```

   正式生成报告时必须带 `--use-region-api`，让生成出的初版报告已经补齐 `项目区域概况`；不得先生成缺概况的报告再另行手工补段。

8. 生成检查 XLSX。

   ```bash
   python3 -B 智能生成报告技能资料/工具脚本/check_smart_report.py \
     --docx "生成报告/项目名称-智能生成勘探报告.docx" \
     --form "过程资料/项目名称/人工填写表/项目名称_智能报告信息填报表_确认生成.xlsx" \
     --drawings-dir "基础信息/项目目录/3.执行资料/3.制图成果" \
     --photos-dir "基础信息/项目目录/3.执行资料/1.外业成果" \
     --check-report "过程资料/项目名称/生成检查/项目名称-智能生成勘探报告-生成检查报告.txt" \
     --xlsx-output "生成报告/项目名称-专项检查表.xlsx"
   ```

9. 运行验证。

10. 后续修改与定稿清理。

   - 初版报告生成后，后续修改均以该报告为母版；每次修改前先复制备份到 `过程资料/项目名称/备份/`，再继续修改当前报告。
   - 不使用参考报告、旧报告或基准模板作为后续修改底稿，除非用户明确要求重来。
   - 用户确认定稿后，`生成报告/项目名称/` 只保留定稿报告和最终检查 XLSX；过程检查 txt/md、备份和诊断材料全部放入 `过程资料/`。

   ```bash
   python3 -B 智能生成报告技能资料/工具脚本/run_smart_report_workflow.py doctor
   python3 -B generate_report.py self-test
   ```

## 禁止事项

- 不要默认使用或恢复 `*_人工确认表_精简版.xlsx`。
- 不要把废弃的旧 `*_人工填写表.xlsx` 当作新流程默认表。
- 不要从旧目录 `生成成果/智能模板/填报表模板/` 取模板。
- 不要把报告、检查表或过程文件写入 `基础信息/`。
- 不要在用户确认模板前生成正式报告。
- 不要用填报表内容覆盖公司模板固定信息。

## 公司模板规则

- `勘探单位` 只用于匹配模板。
- 公司名称、人员名单、资质附件、页眉页脚、签章、落款等公司固定内容应保留自选中的智能基准模板。
- 项目事实、图纸、照片、坐标、勘探记录、区域概况、文物概况等内容来自填报表和项目源资料。

## 常见问题

### 直接开始生成了

通常是跳过了填报表确认。应回到人工填写表或完整 `智能报告信息填报表`，等用户确认补全后再继续。

### 选到了精简确认表

这是旧流程残留。应改用完整 `智能报告信息填报表` 风格。

### 找不到模板

先运行：

```bash
python3 -B 智能生成报告技能资料/工具脚本/run_smart_report_workflow.py doctor
```

如果模板矩阵显示缺失，需要补齐对应公司、遗迹状态和分区状态的智能基准模板。旧模板只能作为升级来源，不能直接替代当前智能基准模板。

### 不能做可视化 DOCX 检查

如果电脑没有 Word 或 LibreOffice，不要声称已经做了可视化版式检查。可以说明已完成 DOCX ZIP/XML 结构校验、文本断言、媒体数量检查和自动化测试。

## 完成标准

- 项目源资料文件夹明确。
- 使用完整 `智能报告信息填报表` 风格。
- 用户确认填报表已补全。
- 推荐模板与 `勘探单位`、遗迹状态、分区状态匹配。
- 用户确认模板。
- 输出报告/计划和一个检查 XLSX 位于 `生成报告/`；过程检查 txt/md 不留在最终成果文件夹。
- 相关验证通过，或明确说明未通过项和原因。
