# 05-报告拼装与版式模块

## 定位

报告拼装与版式模块只拼装和排版，不新增事实、不重做分析；正式正文生成前必须清除待复核、待确认、待核验、待补充、未确认、未写明、资料缺失等占位表述。

第一章“评估项目基础信息”不得写成 AI 或脚本的工作过程说明。若 `quote_candidates.jsonl` 中存在来自文物调查报告、核查报告、考古调查勘探报告或主管部门函件的项目基础事实段，应优先作为该小节底稿；只能按文评行文需要做轻度衔接，不得写“根据《XX报告》”“XX报告中提到”等项目资料出处引导语，不得替换为“本报告重新整理、OCR、事实抽取和影响识别”等过程自述。

项目基础事实段之后必须套用固定影响评估引入段：“现结合项目当前地理位置、施工方案……”至“需系统性评估【项目名称】在建设及运营期间对所涉及文物的影响。”。该段只替换项目名称和文保点/文物名称，不得改写成文评定义、法规通知或方法说明。

第一章第一节下固定两个小标题：`1.评估项目基础信息` 和 `2.文物影响评估必要性`。`2.文物影响评估必要性` 下必须写入固定四段：文物影响评估定义段、《文物保护法》第二十八条、第二十九条、内政办发〔2024〕40号和内文物发〔2025〕6号依据段；四段均不做任何替换修改。

第一章“（三）评估内容”固定收束句只替换项目名称，原文为：“根据文物影响的程度，明确【项目名称】是否可行，提出调整建议和针对性的减缓措施。并根据【项目名称】的文物影响评估结论，提出其他相关建议。” 不得改写为复核问题、最终等级或审批结论说明。

第一章“（五）评估依据 / 1.宪章公约”为固定内容和固定格式，必须逐条编号列出（1）至（6）六项宪章公约；不得合并为“1.宪章公约：……”一行，不得删除英文括注、年份、编号或标点。

第一章“（五）评估依据 / 2.法律法规”前三条为固定底座，必须逐条编号列出《中华人民共和国文物保护法》《中华人民共和国文物保护法实施条例》《内蒙古自治区文物保护条例》并写明修订/施行时间。是否补充《长城保护条例》等其他法规、保护条例，应依据所涉及文保点和项目资料判断；凡新增补充项，必须同样写明修订或施行时间，不得只列文件名。

第一章“（五）评估依据 / 3.文件规定”为固定内容和固定版式，必须按（1）至（7）逐条列出七项文件规定，编号后不加空格；不得改成“政策文件”，不得合并为“3.文件规定：……”一行，不得删改文号、日期或标点。

第一章“（五）评估依据 / 4.技术规范”只固定标题层级，正文通常应从本项目可研报告、建设方案、设计文件或施工方案中的对应章节完整摘录。不得随意修改、概述、选择性摘抄，不得用“本项目资料明确引用……具体清单需进一步核验”等概述句替代；未找到对应章节且本节必需时作为阻断缺项处理，非必需条目略过，不在正文写待核验提示。

第一章“（五）评估依据 / 5.其他资料”前两条固定为《世界文化遗产影响评估指南》（2011年）和《中国文物古迹保护准则》（2015年）。其他资料根据本项目资料补入，主要为项目相关报告、保护规划、城乡规划等规划文件；不得把各类图纸、图片、正射影像、红线、KML/OVKML、坐标表、现场照片、验收意见书等文件列入该小节。

第七章为固定内容，仅作变量替换。报告拼装模块只根据分析模块输出的项目方案合规状态选择第七章第一段模板：若项目方案均合规，固定写“基于上述分析，本项目建设及运营过程对涉及的【文物名称】的影响相对轻微。项目设计符合法律法规和相关规范，具有可行性。”；若存在须调整后方可合规的内容，固定写“基于上述分析，本项目建设及运营过程对涉及的【文物名称】【涉及区划】的影响相对轻微。但须全面落实保护措施，将【控制对象】严格控制在【控制要求】。调整后，项目设计符合法律法规和相关规范，具有可行性。”。之后固定写现行工程方案重评触发句、系统实施减缓措施后的控制结论、长效监测评估机制和“其他建议”两段，不得自由发挥或另写审批式结论。

第八章“支撑法律法规及文件”内容及格式完全固定，必须使用 `assets/07-文物影响评估报告正文模板.md` 和 `references/11-固定正文结构与固定内容.md` 中的六项法规政策及条文全文。报告拼装模块不得删减、摘要、改写、追加专项法规、保护规划、技术规范或写“与本项目关系”分析。只有相关法律法规、政策文件更新版本时，才允许整体更新第八章固定文本。

## 必用模板资源

正式文评报告拼装时，优先使用完整模板资源，不得只使用简化基础模板：

```text
assets/07-文物影响评估报告正文模板.md
assets/14-附表综合评估大表模板.md
assets/11-信息入文章节映射表模板.md
assets/13-第五章影响评估表模板.md
assets/09-docx成稿自检清单模板.md
```

`assets/报告草稿基础模板.md` 只允许用于快速烟测或资料严重不足时的临时摘要，不得作为正式文评报告的默认正文模板。

默认正式报告章法为：

```text
一、总则
二、建设项目涉及文物概况
三、建设项目规划概况
四、项目用地范围与文物空间分布关系
五、建设项目可能对文物造成的影响分析与评估
六、减缓措施建议
七、文物影响评估结论及建议
八、支撑法律法规及文件
```

## 阶段脚本

正式 v0.2.2 报告拼装入口：

```bash
python scripts/run_report_assembly.py --workspace "<资料处理输出目录>"
```

脚本只负责把事实段、分析段、矩阵表、证据映射和附件索引拼入报告，不新增事实、不重做分析。输出必须至少形成完整八章结构、影响识别矩阵、保护措施矩阵、干净版和证据版 Word。

最终交付目标是一篇像参考成稿那样图文并茂、表格准确、章节完整的正式报告。表格结构和 Word 基础版式已经由 `assets/` 与 `references/10-docx成稿规则.md`、`references/14-Word字体与版式迁移要求.md` 规定，阶段脚本必须执行这些既有规范，不得把它们当作后续再设计事项。复杂图纸、照片、位置关系图等图件自动选图和图文视觉 QA，依据 `processing_output/figure_index.jsonl` 和 `extracted_figures/` 继续复核，不得把无图的摘要稿视为最终成稿。

v0.2.2 额外要求：项目位置图、总平面图、用地范围图、正射影像图、保护区划图、空间关系图、距离图、视线或景观分析图必须优先从项目图纸、坐标/GIS、文物保护规划、文物部门资料或专门制图中选取。勘探报告图件只可用于勘探工作说明、地下文物风险判断和程序依据，不得默认替代文评正文图件体系。

v0.2.2 模板额外要求：`4.技术规范` 只固定标题层级，条目内容通常完整摘自本项目可研报告、工程方案、设计说明、施工方案等资料中的对应章节，不得概述或选择性摘抄；`5.其他资料` 前两条固定，后续只根据本项目资料写入项目相关报告、规划文件，不纳入图纸、图片、正射影像、红线、KML/OVKML、坐标表、现场照片、验收意见书；`（六）评估方法` 全文固定，仅替换项目名称和文物名称，不得摘要化。

第二章标题下必须保留开头总起段。若 `quote_candidates.jsonl` 中存在 `quote_type=第二章开头总起段`，应优先写入该段，再写文物对象承接句和后续小节；不得直接从“二、建设项目涉及文物概况”跳到“（一）文物所在区域概况”。

第五章标题下必须保留固定两段总起段，仅替换项目名称和文物名称，并且必须位于 `（一）项目方案设计合规性评估` 之前；不得删除、改写、缩减、移动到小节内，也不得用影响识别矩阵说明或 AI 概述替代。

第二章“（一）文物所在区域概况”下必须分为 `1.地理位置`、`2.自然概况`、`3.社会经济情况`、`4.历史沿革` 四个小节。各小节正文一般来自网页搜索；若当前阶段尚未联网检索，应保留四个小节并写待联网检索提示，不得合并成一个区域概况总段。

第二章“（二）【文物名称】概述”下必须固定六个小节：`1.遗址基本情况`、`2.遗址现状评估`、`3.调查、发掘、保护工程情况`、`4.保护范围及建设控制地带`、`5.价值陈述`、`6.价值评估`。本节内容一般来自文物调查回函、文物调查报告、文物核查报告或主管部门函件；未抽取到正文时保留小节并标注待依据调查类材料补充。`6.价值评估` 中固定说明段、`表1 遗产价值分级量表` 和表格必须完整写入，表后评价段只能替换文物名称、文物保护单位级别、价值方面和等级。

第五章三张正文表的表名固定为 `项目设计合规性评估`、`项目建设期影响评估`、`项目运营期影响评估`，表号不固定，由最终报告顺序生成。报告拼装阶段只能根据 `analysis/chapter5_design_compliance.jsonl`、`analysis/chapter5_construction_impacts.jsonl`、`analysis/chapter5_operation_impacts.jsonl` 和 `assets/13-第五章影响评估表模板.md` 填表，不得新增行列、改名或自造影响因子。`项目建设期影响评估`、`项目运营期影响评估` 的三类影响程度只能使用固定值：`有较大益处`、`有较小益处`、`正面影响可忽略`、`没有改变`、`负面影响可忽略`、`有较小负面影响`、`有较大负面影响`。

第五章评估表格不能替代正文。报告拼装时必须先根据分析阶段输出的自然段分析要点、影响路径、项目事实和对应条文写正文，再放固定评估表：

- `（二）项目建设期影响评估`：先写施工期环境影响、地下不明文物扰动等自然段，说明影响来源、影响路径、控制条件和初步判断理由，再放建设期影响评估表。
- `（三）项目运营期影响评估`：先写污染排放与生态环境影响、整体建筑风貌与视觉景观影响等自然段，再放运营期影响评估表。
- `（四）文物影响综合评估`：先综合遗产资产价值、建设期和运营期影响、减缓措施前提，再写固定的《世界文化遗产影响评估指南》影响评估说明，插入 `assets/16-世界文化遗产影响评估指南-影响评估量表.png` 和固定图题，随后写表前承接句，最后放综合影响评估大表。

不得生成只有一句概述加表格的第五章；评估表只作为归纳及程度分析，不承担正文论证功能。

第五章第四节综合影响评估大表前的固定内容不得省略。固定顺序为：

1. 《世界文化遗产影响评估指南》提出“开发项目或其他改变……”固定段。
2. “本次文物影响评估依照《世界文化遗产影响评估指南》中的影响评估表……”固定段，只替换图号。
3. 固定图片 `assets/16-世界文化遗产影响评估指南-影响评估量表.png`。
4. 固定图题 `图【影响评估量表图号】 《世界文化遗产影响评估指南》影响评估量表`。
5. “基于上述分析，结合【文物名称】的遗产价值……”表前承接句，只替换文物名称、总体结果和表号。
6. 综合影响评估大表。

第三章和第四章中的项目建设必要性、建设及运营方案、选址分析应优先使用 `quote_candidates.jsonl` 中对应的可研、建设方案或项目说明原文章节。可以为文评行文做必要衔接，但不得压缩成短概述，也不得用开工请示、图纸、空间关系或文物保护约束自行代写。

第六章减缓措施必须逐项回应第五章已经识别的影响因素。报告拼装时应优先消费 `analysis/mitigation_matrix.jsonl` 中的影响路径、措施方向和可研/工程方案已有措施，写成成段措施；不得用保护措施矩阵、减缓措施表或其他表格替代第六章正文，不得只输出“复核高度、优化设计”“加强管理、落实监测”等短句。若某项措施来源于可研、工程方案、施工组织或运营方案，应写明按项目前期方案落实的具体措施内容。

## 双版本规则

干净版：

- 文件名：`report_clean.md`、`report_clean.docx`。
- 面向人工审阅、后续转 Word 和继续修改。
- 正文不显示密集证据编号。
- 正文不保留占位式缺项标记。

证据版：

- 文件名：`report_with_evidence.md`、`report_with_evidence.docx`。
- 面向内部核查。
- 可保留证据编号和可回溯来源信息；来源信息用于内部审阅，不把项目资料出处引导语写成正式正文表述。

证据映射表：

```text
section_id
paragraph_id
text_excerpt
fact_ids
evidence_ids
source_files
confirmation_status
```

## Word 基础版式

v0.2.2 负责执行既有 Word 基础版式规则。版式规则来源为 `references/10-docx成稿规则.md` 和 `references/14-Word字体与版式迁移要求.md`，不是由报告拼装脚本重新发明。

必须做到：

- 标题层级清楚。
- 正文段落基本统一。
- 影响识别矩阵、保护措施矩阵等表格能放进正文。
- 图件、照片、图纸页和空间关系资料至少能通过图件索引进入报告附件或插图候选清单。
- 附件编号、图表编号基本连续。
- 占位式缺项表述在正式正文生成前被清除；初步判断不得被改写为最终审批结论。
- 能输出 `.docx` 草稿。

暂不强求：

- 完全符合某个正式单位模板。
- 自动生成精美目录、页眉页脚、封面。
- 图纸、坐标图、复杂插图的精细排版。
- Word 里复杂跨页表格的美化。
