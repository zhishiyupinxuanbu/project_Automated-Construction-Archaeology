# 19-第五章人工预填表与影响因子规则

## 定位

v0.3.1 延续第五章影响评估“人工预填一张表，AI 映射回三张正文表”的模式，并补充预填表驱动的定向条文抽取规则。

用户给项目资料并要求生成文物影响评估报告时，skill 默认先生成并返回 `第五章人工预填表.xlsx`，暂停等待用户填写。用户填完 Excel，或在对话中补充同等内容后，才继续跑报告。

## 人工预填表

人工预填表固定 9 行 6 列，模板见 `assets/15-第五章人工预填表模板.md`。

固定列：

```text
编号
对应正文表
固定影响内容
人工预填说明
依据原文或资料来源
人工边界提示
```

固定行：

| 编号 | 对应正文表 | 固定影响内容 |
| --- | --- | --- |
| 1 | 项目设计合规性评估 | 项目流程 |
| 2 | 项目设计合规性评估 | 环境保护 |
| 3 | 项目设计合规性评估 | 景观保护 |
| 4 | 项目设计合规性评估 | 用地类型 |
| 5 | 项目设计合规性评估 | 建筑高度与建筑密度 |
| 6 | 项目建设期影响评估 | 施工过程对环境的影响（包括污染排放、地质灾害、水土流失、生态环境） |
| 7 | 项目建设期影响评估 | 施工过程对地下不明文物的扰动 |
| 8 | 项目运营期影响评估 | 污染排放与生态环境影响 |
| 9 | 项目运营期影响评估 | 整体建筑风貌与视觉景观影响 |

用户只需填写 `人工预填说明` 和 `依据原文或资料来源`。`人工边界提示` 可以为空。

## AI 处理边界

- 人工预填表是第五章分析判断的优先输入。
- 用户可只列原始事实和法规依据，不必写计算结果、合规结论或影响程度。
- AI 负责整理、计算、比对和初步判断；但常规项目事实、文物事实和空间事实仍由事实抽取模块按全量资料正常抽取。
- 当需要法规、规划、保护要求等具体条文支撑预填表中的项目事实时，事实抽取模块应根据预填表定向抽取对应条文，输出“项目事实 + 对应条文”，分析判断模块再做比对结论。
- AI 不得新增、删除或改名正文表的行列。
- AI 不得自造影响因子表述。
- 证据不足时，不得硬填正文表；应输出“需人工补充项”。

## 项目设计合规性评估映射规则

固定表名：`项目设计合规性评估`。表号不固定，由最终报告顺序生成。

固定列：

```text
建设阶段
影响内容
详细信息
《文物保护法》及《长城保护规划》规定
是否合法合规
```

固定行：

```text
方案阶段 | 项目流程
方案阶段 | 环境保护
方案阶段 | 景观保护
方案阶段 | 用地类型
方案阶段 | 建筑高度与建筑密度
```

映射：

- `固定影响内容` -> `影响内容`
- `人工预填说明` -> `详细信息`
- `依据原文或资料来源` 和事实抽取模块输出的具体条文 -> `《文物保护法》及《长城保护规划》规定`
- AI 根据事实与依据计算、比对后填写 `是否合法合规`

如果预填表写明依据是《长城保护规划》或其他保护规划，不得误写为文物调查回函。表格依据列应写具体条款、章节或控制要求，不能只写“根据《长城保护规划》”。

`是否合法合规` 只能填：

```text
符合
不符合
```

如果证据不足以判断，不得在正式表中写“待确认”；应列为“需人工补充项”，待补充后再成表。

### AI 必须执行的计算和比对

当人工预填只给出原始事实时，AI 应自动补全可审计计算。例如：

- 已知各单体建筑面积合计和扩建后总占地面积时，计算建筑密度。
- 已知建筑高度和规划限高时，比对是否超高。
- 已知项目距离、是否涉及保护范围/建设控制地带和管控要求时，比对是否触发程序或空间管控。

示例：

```text
人工预填说明：扩建总建筑面积7386.00平方米；扩建后总占地面积16919.64平方米；各单体建筑面积合计5115.65平方米；鼓楼拟建三层，总高12米；其余建筑高度均不超过9米。
依据原文或资料来源：建设控制地带内新建或改建建筑总高度不得超过9米，建筑密度不得大于40%。
AI处理：建筑密度=5115.65/16919.64≈30.23%，密度符合；鼓楼12米超过9米限高，高度不符合；`是否合法合规` 填“不符合”。
```

## 建设期和运营期影响评估映射规则

固定表名：`项目建设期影响评估`。

固定表名：`项目运营期影响评估`。

表号不固定，由最终报告顺序生成。

固定列：

```text
建设阶段
影响内容
详细信息
对遗址本体的影响及程度
对环境风貌的影响及程度
对地下文物的影响及程度
```

`项目建设期影响评估` 固定行：

```text
施工阶段 | 施工过程对环境的影响（包括污染排放、地质灾害、水土流失、生态环境）
施工阶段 | 施工过程对地下不明文物的扰动
```

`项目运营期影响评估` 固定行：

```text
运营阶段 | 污染排放与生态环境影响
运营阶段 | 整体建筑风貌与视觉景观影响
```

映射：

- `固定影响内容` -> `影响内容`
- `人工预填说明` -> `详细信息`
- AI 根据 `人工预填说明`、`依据原文或资料来源`、`人工边界提示` 和项目事实，填写三类影响程度。

## 固定影响因子

`项目建设期影响评估`、`项目运营期影响评估` 的三类影响程度只能从下列固定值中选择：

```text
有较大益处
有较小益处
正面影响可忽略
没有改变
负面影响可忽略
有较小负面影响
有较大负面影响
```

不得写：

```text
影响轻微
影响较小
基本无影响
可控影响
较低风险
待评估
待确认
不涉及
```

## 影响因子判断尺度

| 固定影响因子 | 使用条件 |
| --- | --- |
| 有较大益处 | 明确改善遗址本体保存、展示、环境或管理秩序，且作用显著 |
| 有较小益处 | 对环境恢复、管理秩序、展示利用有一定改善 |
| 正面影响可忽略 | 理论上有正面作用，但很弱，不足以作为主要评价 |
| 没有改变 | 对该对象没有明确影响路径 |
| 负面影响可忽略 | 有潜在不利影响路径，但距离、强度或控制措施使影响很低 |
| 有较小负面影响 | 确有不利影响，但范围小、程度轻、可通过措施控制 |
| 有较大负面影响 | 明显冲突、直接扰动、不可忽略，或现有措施不足 |

一般项目的 AI 初步判断最多到 `有较小益处` 或 `有较小负面影响`。只有人工明确说明，或项目事实和管控依据显示存在直接、显著影响时，才允许使用 `有较大益处` 或 `有较大负面影响`。

## 人工边界提示

`人工边界提示` 允许为空。若用户填写，应优先遵守。常见提示包括：

- 需计算景观廊道。
- 鼓楼高度按不符合处理。
- 不要写最终可行性结论。
- 该项只作初步判断，需专业负责人确认。

若人工边界提示与项目证据或法规明显冲突，应列入 `analysis/risk_flags.jsonl`，不得静默采用。

## 输出要求

分析判断阶段应将预填表解析为机器文件，建议输出：

```text
analysis/chapter5_prefill_rows.jsonl
facts/requirement_facts.jsonl
analysis/chapter5_table6_design_compliance.jsonl
analysis/chapter5_table7_construction_impacts.jsonl
analysis/chapter5_table8_operation_impacts.jsonl
analysis/chapter5_prefill_issues.jsonl
```

报告拼装阶段只能根据上述机器文件和 `assets/13-第五章影响评估表模板.md` 写入三张固定表，不得重新发明表结构。

第五章评估表格不能替代正文。分析判断阶段应把每项影响的自然段分析要点一并输出，报告拼装阶段先写正文分析，再放固定评估表。
