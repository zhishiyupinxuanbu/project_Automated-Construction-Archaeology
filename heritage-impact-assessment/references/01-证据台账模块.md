# 01-证据台账模块

## 定位

证据台账是所有模块之间的交接单。它回答三件事：

- 信息来自哪里。
- 信息能不能用、怎么用。
- 信息应该进入报告哪一部分。

## 产物

```text
evidence/
├── evidence_register.csv
├── evidence_register.jsonl
└── section_evidence_index.jsonl
```

## 底层证据表

一条独立事实一条证据，不把一整段可研或整页报告塞成一条。

必备字段：

```text
evidence_id
source_file_id
source_file
source_location
evidence_type
original_excerpt
structured_value
use_mode
applicable_section
confidence
confirmation_reason
```

红头文件、主管部门函件、规划批复和其他正式文件还应登记以下元信息。缺失时记录为空并写入 `notes`，不得在台账阶段猜测补全：

```text
issuing_authority
document_title
document_number
document_date
body_page_range
attachment_page_range
```

`use_mode` 取值：

```text
可直接引用
需改写
只可参考
待确认
不可引用
```

## 章节证据索引

章节证据索引用于报告拼装，不重复保存长原文，只引用证据编号。

字段：

```text
section_id
section_title
evidence_id
fact_id
recommended_use
notes
```

## 使用规则

- 没有 `evidence_id` 的事实不得进入报告正文。
- 联网来源必须进入证据台账和外部依据清单。
- 证据台账不做专业影响判断，不抽取保护要求，不改写条文内容。
- 证据冲突不在台账阶段消解，交给事实抽取模块做来源优先级和冲突保留。
- 文号识别不完整、正文页/附件页边界不清时，只记录内部问题；不得把残缺文号或不完整正文页直接交给报告拼装。
