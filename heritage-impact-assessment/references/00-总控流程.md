# 00-总控流程

## 目标

总控只负责选择模块、检查阻断缺项、串联机器产物和控制输出。不要在总控阶段通读全部原始资料或直接写报告。

v0.3.1 支持任务级硬隔离：每个模块可以在新上下文中启动，依靠上游机器产物和下一阶段启动提示继续工作。

v0.3.1 延续第五章人工预填闸门：用户给到项目资料并要求生成文物影响评估报告时，默认先生成并返回 `第五章人工预填表.xlsx`，暂停等待用户填写或用对话补充同等内容。用户填完后，skill 再继续跑资料处理、事实抽取、分析判断和报告拼装。

## 默认流程

```text
用户给项目资料包
→ 生成第五章人工预填表.xlsx
→ 暂停，等待用户填写或对话补充
→ 将预填表纳入 human_input/
→ 资料处理模块
→ 检查不可替代资料缺项
   → 有：暂停，只告诉用户必须补什么
   → 无：继续
→ 证据台账模块
→ 事实抽取模块
→ 分析判断模块
→ 报告拼装与版式模块
→ 输出干净版 Word + 证据版 Word
```

硬隔离运行时，流程改为：

```text
阶段 0：第五章人工预填
→ 读取 references/19-第五章人工预填表与影响因子规则.md
→ 运行 scripts/generate_chapter5_prefill_template.py
→ 返回 第五章人工预填表.xlsx 并暂停

阶段 A：资料处理
→ 写 run_state/material_processing.module_done.json
→ 写 next_prompts/next_prompt_fact_extraction.md

阶段 B：事实抽取
→ 新上下文只读 next_prompt_fact_extraction.md 和上游产物
→ 写 run_state/fact_extraction.module_done.json
→ 写 next_prompts/next_prompt_analysis.md

阶段 C：分析判断
→ 新上下文只读 next_prompt_analysis.md、human_input/第五章人工预填表 和上游产物
→ 写 run_state/analysis.module_done.json
→ 写 next_prompts/next_prompt_report_assembly.md

阶段 D：报告拼装
→ 新上下文只读 next_prompt_report_assembly.md 和上游产物
→ 写最终报告和 run_state/report_assembly.module_done.json
```

## 上下文控制

- 每个模块只读取自己的规则文件和上游机器产物。
- 第五章人工预填表是人工输入，不属于聊天上下文；用户通过对话补充的内容必须整理进 `human_input/chapter5_prefill_notes.md` 或等效机器文件后再进入后续阶段。
- 不得把整包 OCR 文本、整份可研、整份报告一次性塞进上下文。
- 需要原文时，先读索引，再按证据编号或文件编号读取必要片段。
- 模块之间通过 `jsonl`、`csv`、`txt`、`geojson`、`docx` 交接。
- 阶段之间不得依赖聊天历史；下一阶段必须能只凭 `module_state.json`、`module_done.json`、`next_prompt_*.md` 和上游产物继续。
- 若当前上下文已经积累大量 OCR、原文或分析过程，必须停止当前阶段并用下一阶段提示另起上下文。

## 暂停条件

默认首次暂停发生在第五章人工预填闸门：返回 Excel 模板，等待用户填写。

进入资料处理后，只有 `processing_output/user_blocking_gaps.jsonl` 非空时暂停。暂停时只输出不可替代缺项，不输出内部异常。

不可替代缺项是指：项目包、可替代资料、长期知识库和官方联网来源均无法补足，且会影响以下事项：

- 项目基本事实。
- 项目空间边界。
- 文物对象身份。
- 核心管控依据。

## 不暂停情形

- 没有保护规划，但官方网站或长期知识库能补足。
- 没有工程方案或设计方案，但可研、项目建议书或批复能支撑建设内容。
- 没有既有文评、保护方案、评审意见。
- 合同、询价函、营业执照、法人身份证不完整。
- OCR 质量一般但仍能抽取关键事实。

## 默认成果

- `report_clean.docx`
- `report_with_evidence.docx`
- `report_clean.md`
- `report_with_evidence.md`
- `report_evidence_map.jsonl`

内部成果保留在项目工作包，不主动展示给用户。

## v0.3.1 第五章人工预填规则

- 预填表只有一张，固定 9 行 6 列：`编号`、`对应正文表`、`固定影响内容`、`人工预填说明`、`依据原文或资料来源`、`人工边界提示`。
- 用户可只填写 `人工预填说明` 和 `依据原文或资料来源`；`人工边界提示` 可以为空。
- 项目事实、文物事实和空间事实仍由事实抽取模块正常抽取；只有抽取法规、规划、保护要求等条文时，才根据预填表和已抽项目事实定向抽取对应条文。
- AI 必须基于人工预填事实、依据和边界提示，计算、比对并映射回第五章三张固定正文表。
- `项目设计合规性评估`、`项目建设期影响评估`、`项目运营期影响评估` 的表名固定；表号不固定，由最终报告顺序生成；横向字段和纵向影响内容固定，不得新增、删减、改名。
- `项目建设期影响评估`、`项目运营期影响评估` 的影响程度只能使用固定影响因子：`有较大益处`、`有较小益处`、`正面影响可忽略`、`没有改变`、`负面影响可忽略`、`有较小负面影响`、`有较大负面影响`。
- 一般项目的 AI 初步判断最多到 `有较小益处` 或 `有较小负面影响`；只有人工明确说明或证据显示直接显著影响时，才允许使用 `有较大益处` 或 `有较大负面影响`。
- 第五章评估表格不能替代正文；报告拼装必须先写自然段影响分析，再放固定评估表。

## v0.2.2 成稿固定闸门

正式文评成稿不得停留在摘要式草稿。进入报告拼装、成稿复核或用户指出“固定内容缺失、被缩减、图件来源错误”时，必须读取：

```text
references/15-成稿固定规则与图件来源.md
references/16-两份成稿完全相同字符串模板.md
references/17-两份成稿相似句变量模板.md
references/18-成稿固定内容两层模板模型.md
```

总控需确认后续阶段执行以下硬边界：

- 第一章六节、第二章区域和文物概况、第三章项目必要性与建设运营方案、第四章空间关系与选址分析、第五章正文分析加表格、第六章减缓措施、第七章其他建议、第八章法规支撑和附表均按觉海寺、正蓝旗成稿共同章法保留。
- `16-两份成稿完全相同字符串模板.md` 中的标题和正文段落为硬模板原文，正式成稿必须一字不差保留。
- `17-两份成稿相似句变量模板.md` 中的变量模板句按占位逻辑替换项目名称、文物名称、地区、文号、日期、数量等变量，不得改写句法骨架；该文件以“截取修正版”为准。
- `18-成稿固定内容两层模板模型.md` 中的同类变量内容必须按同等层级和同等展开密度写成本项目事实。
- 固定正文、固定表格、遗产价值分级量表、第五章三类评估表、文物影响评估量表和附表一不得被摘要化或静默删除。
- 图件按文评图件体系独立取源；项目位置图、总平面图、用地范围图、空间关系图、距离图、正射影像、保护区划图等不得默认从勘探报告截图替代。
- 勘探报告图件不插入第四章第三节；至多服务第五章地下文物风险判断和程序依据。
