# 04-分析判断模块

## 定位

分析判断模块只基于事实抽取成果、证据台账、官方依据和低权重案例参考进行分析。它不得重新通读整包原始资料。

## 输出

```text
analysis/
├── impact_matrix.jsonl
├── mitigation_matrix.jsonl
├── chapter5_fact_rule_pairs.jsonl
├── chapter5_design_compliance.jsonl
├── chapter5_construction_impacts.jsonl
├── chapter5_operation_impacts.jsonl
├── chapter5_pairing_issues.jsonl
├── risk_flags.jsonl
├── analysis_notes.jsonl
└── expert_review_points.jsonl
```

## 阶段脚本

正式 v0.2.2 分析判断入口：

```bash
python scripts/run_analysis.py --workspace "<资料处理输出目录>"
```

脚本只读 `facts/`、`evidence/`、`processing_output/external_sources.jsonl`、`human_input/` 和阶段状态，生成第五章固定表机器底稿、影响识别矩阵、保护措施矩阵、风险提示、专家审核关注点、`run_state/analysis.module_done.json` 和 `next_prompts/next_prompt_report_assembly.md`。

脚本输出是报告分析的结构化底稿。涉及第七章固定结论时，分析模块必须明确输出项目方案合规状态、是否需要调整后合规、控制对象和控制要求，供报告拼装模块选择固定模板；若无法确认，不得把“待专业负责人确认”写入正文，而应写入内部问题并在生成正文前阻断。

## 硬隔离输入

分析判断阶段可在新上下文中启动。启动时只读取：

- `next_prompts/next_prompt_analysis.md`
- `module_state.json`
- `facts/project_facts.jsonl`
- `facts/heritage_facts.jsonl`
- `facts/requirement_facts.jsonl`
- `facts/quote_candidates.jsonl`
- `facts/fact_issues.jsonl`
- `evidence/evidence_register.jsonl`
- `processing_output/external_sources.jsonl`
- `human_input/chapter5_fact_rule_pairs.json`、`human_input/chapter5_fact_rule_pairs.csv` 或 `human_input/chapter5_pairing_notes.md`
- 本文件
- `references/19-第五章事实条文匹配页面与影响因子规则.md`

不得读取上一阶段聊天记录。不得重新通读全部原始资料、可研或既有报告。

若人工配对结果引用的条文在 `facts/requirement_facts.jsonl` 中缺失，或某个配对项缺少必要事实、条文、判读口径，分析判断模块不得回到原始文件自行抽取；应写入 `chapter5_pairing_issues.jsonl` 或 `risk_flags.jsonl`，提示事实抽取阶段或人工配对阶段补齐。

## 必用专业 reference

正式文评分析判断时，按需读取分析、价值、空间关系和样本章法规则：

```text
references/05-文评成稿样本与文章架构.md
references/11-固定正文结构与固定内容.md
references/12-文物对象概述与价值评估规则.md
references/13-空间关系写作规则.md
assets/12-遗产价值分级量表模板.md
assets/13-第五章影响评估表模板.md
assets/14-附表综合评估大表模板.md
references/19-第五章事实条文匹配页面与影响因子规则.md
```

这些 reference 用于保证分析不是摘要式判断，而是能支撑正式报告第五章和综合评估表。

## 表格归属

- 原始表格归资料处理模块。
- 事实表格归事实抽取模块。
- 第五章事实条文配对结果归 `human_input/`；分析判断模块只能解析、校验和映射，不得擅自新增人工配对关系。
- 第五章三张固定正文表的机器底稿归分析判断模块。
- 报告拼装模块只负责把分析表格呈现为报告图表，不重新做分析。

## v1.0 第五章事实条文配对结果

分析判断阶段必须优先读取 `human_input/` 中的第五章事实条文配对结果。人工输入可以是：

```text
human_input/chapter5_fact_rule_pairs.json
human_input/chapter5_fact_rule_pairs.csv
human_input/chapter5_pairing_notes.md
```

若用户通过对话补充而非导出 JSON/CSV，应先将对话内容整理成 `human_input/chapter5_pairing_notes.md` 或等效机器文件，再进入分析判断阶段。

配对结果至少包含：

```text
pair_id
fact_id
requirement_id
target_table
impact_content
human_judgment
impact_factor
notes
```

`human_judgment`、`impact_factor` 和 `notes` 可以为空。AI 应根据配对事实、配对条文、人工判读和备注做计算、比对和初步判断。

分析判断模块处理第五章配对结果时，只做“项目事实 + 对应条文/控制要求”的比对，不重新抽条文。示例：

```text
输入项目事实：鼓楼总高12米。
输入对应条文：《长城保护规划》第X章第X条规定，建设控制地带内建筑高度不得超过9米。
分析比对：12米 > 9米。
输出结论：不符合规定。
```

若配对条文来源为《长城保护规划》，表格依据列和正文分析不得误写成文物调查回函；若只取得文件名而无具体条款，不能写空泛的“根据《长城保护规划》”，应列为需补具体条文。

## 第五章固定正文表

第五章正文只生成三张固定表：

```text
项目设计合规性评估
项目建设期影响评估
项目运营期影响评估
```

表号不固定，由最终报告顺序生成。表名、横向字段和纵向影响内容固定，不得新增行列或改名。

`项目建设期影响评估`、`项目运营期影响评估` 的三类影响程度只能使用以下固定影响因子：

```text
有较大益处
有较小益处
正面影响可忽略
没有改变
负面影响可忽略
有较小负面影响
有较大负面影响
```

一般项目的 AI 初步判断最多到 `有较小益处` 或 `有较小负面影响`。只有人工明确说明或资料显示直接、显著影响时，才允许使用 `有较大益处` 或 `有较大负面影响`。

不得自造“影响轻微、基本无影响、可控影响、较低风险”等表述。

第五章评估表格是正文分析后的归纳和程度判断，不能替代正文。分析判断阶段应同时输出可供报告拼装使用的自然段分析要点，至少说明影响来源、影响路径、对应项目事实、对应条文或证据、初步判断理由。

建议新增或补充输出字段：

```text
analysis_text_points
related_requirement_ids
comparison_logic
```

`analysis/chapter5_design_compliance.jsonl` 还应提供第七章固定模板所需字段：

```text
scheme_compliance_status: all_compliant | compliant_after_adjustment | blocking_unknown
control_object
control_requirement
related_requirement_ids
chapter7_template_choice
```

若项目方案均合规，`chapter7_template_choice` 写 `all_compliant`；若存在需调整后方可合规的内容，写 `compliant_after_adjustment`，并填明控制对象和控制要求；若无法确认，写 `blocking_unknown` 并作为正文生成前阻断问题，不在报告正文写待确认。

建议字段：

```text

spatial_relation
phase
impact_pathway
evidence_ids

judgment_limit
mitigation_direction

```

## 初步影响判断

可以给出初步影响判断，但不得给出最终影响等级或最终技术结论。

允许：

- 可能存在景观风貌影响。
- 施工期扰动风险较低，待专业负责人确认。
- 是否涉及缓冲区需进一步核定。

禁止直接写：

- 影响轻微。
- 项目建设可行。
- 符合审批要求。

## 保护措施对应

保护措施建议必须与影响路径一一对应。

保护措施矩阵字段：

```text
mitigation_id
impact_id
heritage_object
phase
risk_source
control_measure
responsible_party
monitoring_or_acceptance
evidence_ids
professional_confirmation_points
```

不得只输出“加强施工管理、做好保护宣传、落实监测”等泛泛措施。

保护措施应优先对应第五章影响路径，并吸收可研、工程方案、施工组织、运营方案中已有的预防、环保、管理、监测和应急措施。若上游事实中没有对应措施来源，可生成措施方向，但必须标注为需专业复核，不得写成已经落实的工程方案。

## 依据权重

分析依据优先级：

1. 本项目证据台账。
2. 官方法规、条例、保护规划、批复、管理要求。
3. 专业方法论和技术路线。
4. 长期知识库中的相似案例、专家意见、历史项目经验。
5. 非本项目的既有文评、保护方案、园区评估。

相似案例和专家意见权重较低，只用于提醒可能漏掉的影响类型、保护措施类型、专家关注点和写法逻辑。不得用“相似案例曾经这样判断”替代“本项目证据显示什么”。

## 阶段硬隔离输出

分析判断阶段结束时必须写入：

```text
run_state/analysis.module_done.json
next_prompts/next_prompt_report_assembly.md
```

`next_prompt_report_assembly.md` 必须说明：

- `analysis/impact_matrix.jsonl` 路径。
- `analysis/mitigation_matrix.jsonl` 路径。
- `analysis/chapter5_design_compliance.jsonl` 路径。
- `analysis/chapter5_construction_impacts.jsonl` 路径。
- `analysis/chapter5_operation_impacts.jsonl` 路径。
- `analysis/risk_flags.jsonl` 路径。
- `facts/` 路径。
- `evidence/section_evidence_index.jsonl` 路径。
- 下一阶段只负责报告拼装和基础 Word 版式，不得新增事实或重做分析。
