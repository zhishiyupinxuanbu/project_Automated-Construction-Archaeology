# 建设项目文物保护方案 v0.1

本技能用于参照既有保护方案成稿和配套文物影响评估报告，把用户给定项目资料整理为可审核的文物保护方案工作包或草稿。保护方案的章节排布通常从文评报告中摘取并重排：文评负责“评估与结论”，保护方案负责“保护措施如何落地”。

## 总原则

- 全部用户可见内容使用中文；路径、命令、代码标识可保留英文。
- 默认把用户提供的项目资料文件夹作为事实来源；不要把具体项目事实写入长期知识库。
- 不编造法规、文号、坐标、面积、距离、批复、保护范围、建设控制地带、专家意见或仪器参数。
- 不擅自作最终技术结论；涉及保护范围划定、工程避让距离、煤柱留设、发掘前置、振动限值等高风险内容，必须标记为【待专业负责人确认】。
- 保护措施必须和影响问题、文物对象、工程阶段、责任主体、验收或监测方式对应，不能只写原则性口号。
- 文评报告与保护方案同时存在时，必须先按 `references/07-文评到保护方案摘取规则.md` 建立摘取映射，再写保护方案正文。
- 参考成稿只用于章法、表达、附件结构和措施类型参考，不得套用样本项目事实。

## 工作模式

根据用户意图选择一种模式。

### 1. 启动保护方案

当用户说“写保护方案”“做文物保护方案”“根据资料出方案”时：

1. 读取 `references/01-工作流.md`。
2. 读取 `references/06-参考报告目录与文章架构.md`，先确认本次项目更接近小型工程、公路、煤矿/采矿、建筑改扩建或复杂多文物对象方案。
3. 若用户给出项目资料夹，先运行 `scripts/scan_project_folder.py` 生成资料盘点；若没有资料夹，直接输出资料缺项清单。
4. 使用 `assets/01-资料缺项清单模板.md`、`assets/02-项目信息提取表模板.md` 和 `assets/03-文物对象与工程关系表模板.md` 建立事实底稿。
5. 只从用户资料或人工确认内容提取项目名称、建设单位、建设内容、项目位置、文物对象、保护级别、空间关系、批复和审查意见。
6. 资料不足时先列缺口，不要直接补写项目事实。

### 2. 从文评摘取保护方案

当用户给出文物影响评估报告，或说明“保护方案从评估里摘出来”时：

1. 读取 `references/07-文评到保护方案摘取规则.md`。
2. 读取文评报告的目录、第二章文物概况、第三章项目概况、第四章空间关系、第五章影响评估、第六章减缓措施建议。
3. 使用 `assets/08-信息入文章节映射表模板.md` 建立“文评章节-保护方案章节-改写方式”。
4. 将文评中的评估性语言改写为保护方案的实施性语言；保留事实，删除或弱化最终评价结论。
5. 若文评缺少措施落实细节，再读取 `references/08-正文模板与写作说明.md` 和 `references/04-保护措施库路由.md` 补成责任、时段、巡查、监测、应急和附件。

### 3. 生成方案草稿

当资料边界基本明确，用户要求“生成草稿”“出正文”“形成方案”时：

1. 读取 `references/03-文章结构与写作边界.md`、`references/06-参考报告目录与文章架构.md` 和 `references/08-正文模板与写作说明.md`。
2. 使用 `assets/04-保护措施矩阵模板.md` 先形成措施矩阵，再生成正文。
3. 使用 `assets/05-保护方案正文模板.md` 写正文，而不是只给大纲。
4. 默认正文结构为：总则、项目基本情况、文物概况、文物保护措施、资金或责任保障、附件附表；复杂项目可按施工前、施工期、运营期拆章。
5. 每项措施写明适用阶段、保护对象、风险来源、控制要求、责任主体、监督方式和待确认点。

### 4. 处理专家意见

当用户给出评审意见、修改意见或要求“按专家意见修改保护方案”时：

1. 读取 `references/02-资料识别规则.md` 和 `references/05-Word成稿与附件规则.md`。
2. 将意见拆为保护方案相关、文评报告相关、两者共通、超出资料依据四类。
3. 使用 `assets/06-专家意见修改对照表模板.md` 形成“专家意见-修改位置-修改后内容-待确认”对照。
4. 不把专家意见当作已经核实的项目事实；涉及坐标、距离、煤柱、发掘、监测点位等内容必须回到原始资料或人工确认。

### 5. 复核与交付

当用户要求“检查方案”“自检”“生成最终包”时：

1. 读取 `references/05-Word成稿与附件规则.md`。
2. 使用 `assets/07-方案成稿自检清单模板.md` 检查标题、目录、图表目录、正文结构、附件附表、事实来源、措施闭环和待确认项。
3. 若需要工作包，运行 `scripts/generate_protection_bundle.py`。
4. 默认交付 Markdown 草稿、资料清单、措施矩阵、专家意见对照和自检清单；正式 Word 成稿需用户明确要求并经人工复核。

## 资源路由

- `references/01-工作流.md`：每次启动方案任务都读。
- `references/02-资料识别规则.md`：需要盘点项目资料、识别文评报告、审查意见、图纸、坐标、勘探报告、批复时读。
- `references/03-文章结构与写作边界.md`：需要写正文、改大纲或判断章节是否齐全时读。
- `references/04-保护措施库路由.md`：需要选择施工前、施工期、运营期、应急、煤矿、公路、景观风貌、环境污染、人员管理等措施时读。
- `references/05-Word成稿与附件规则.md`：需要形成 Word、附件、附表、图表目录或专家修改对照时读。
- `references/06-参考报告目录与文章架构.md`：需要参照既有保护方案样本、判断保护方案章法时读。
- `references/07-文评到保护方案摘取规则.md`：有文评报告或需要从文评生成保护方案时必须读。
- `references/08-正文模板与写作说明.md`：需要写保护方案正文、把评估性语言改成实施性语言时读。
- `references/09-原文溯源索引.md`：需要核对参考成稿、源文件和章节来源时读。
- 外部资料库 `05_报告与方案样本/`：保存参考保护方案、配套文评报告和评审意见；需要核对原文时按 `references/09-原文溯源索引.md` 路由读取。
- `scripts/scan_project_folder.py`：用户给出项目资料夹时运行，生成资料盘点。
- `scripts/generate_protection_bundle.py`：需要初始化固定格式工作包时运行。
- `assets/`：可复制、可填写、可转成 Word 的工作模板。

## 固定输出结构

除非用户只问一个局部问题，输出都保留以下字段：

```text
【输入摘要】
【已依据资料】
【执行步骤】
【产物清单】
【待确认事项】
【风险/待补充】
【人工确认点】
```

生成保护措施时使用：

```text
【保护对象】
【影响或风险来源】
【适用阶段】
【控制措施】
【责任主体】
【监测/巡查/验收方式】
【依据资料】
【待确认事项】
```

## 禁止清单

- 不用“影响评估助手包”的目录结构替代用户给定的项目资料结构。
- 不把样本项目事实套到新项目。
- 不把“景观保护措施”机械保留；若专家要求删除或项目类型不适用，必须删减或标注不适用。
- 不在缺少资料时生成坐标表、围护带、煤柱边界、监测点位或安全距离数值。
- 不省略施工交底、人员培训、现场围挡警示、巡查记录、应急预案、责任主体等落实性内容。
