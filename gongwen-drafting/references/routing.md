# 文种路由和资料需求

## 入口判断

用户说“帮我写”“帮我拟”“帮我生成”“整理成请示”“出一版公文”“修改这个公文”“调格式”等，并指向文物手续、许可申请、核查、勘探、验收、开工或相关 Word 文件时，视为启动本 skill。

用户明确指向“文物调查报告”“市级调查报告”“自治区调查报告”“关于XX项目文物调查的报告”时，不使用本 skill，转用独立 `gongwen-survey-report` skill。

优先处理建设单位、项目单位、企业发出的外部报送文件。不要默认拟文物部门向上级主管部门的内部转报请示。

## 基本判断顺序

1. 识别办理阶段：核查/许可、勘探计划备案、申请开展勘探、支持勘探、勘探报告备案、勘探验收、开工、发掘。
2. 识别发文主体：建设单位、项目单位、企业、文物考古机构、文旅/文物部门。
3. 识别受文单位：优先依据模板抬头、已有复函或审批机关、项目所在地、流程说明、附件清单和用户指令。
4. 识别地区模板：先查 `0.资料索引/02-模板索引.md`、`0.资料索引/文种手册/` 和 `0.资料索引/模板内化/`。
5. 识别必要附件：按文种、模板和实际材料判断，不机械套通用清单。

生成前可用确定性门禁检查地区模板路由：

```bash
python3 agent/check_template_route.py --region 库伦旗 --doc-type 文物核查请示
python3 agent/check_template_route.py --region 库伦旗 --doc-type 文物核查请示 --planned-route generic
```

在独立 `gongwen-drafting` skill 包内，对应命令为：

```bash
python3 scripts/gongwen-agent/check_template_route.py --region 库伦旗 --doc-type 文物核查请示
```

如果索引中存在地方模板而计划走通用模板，脚本必须以非 0 状态退出；此时不得继续生成通用模板稿。

## 常见文种路由

- 文物保护许可申请/文物核查请示：加载 `agent_handoff/skills/gongwen-hecha-fixed-format/SKILL.md`；地区模板优先于通用模板。
- 科右前旗文物保护许可申请/行政审批申请：地区模板优先；标题按“关于（单位项目名称）申请用地范围文物保护许可的申请”，受文单位按“科右前旗文化旅游体育局”，正文保留条目式项目情况。
- 锡林郭勒盟/西乌珠穆沁旗文物调查事宜请示：用户说“锡盟”“锡林郭勒盟”“西乌”“西乌珠穆沁旗”，且文种为文物保护许可申请、文物核查请示或申请办理文物调查事宜时，均优先使用 `模板——锡林郭勒盟西乌珠穆沁旗文物调查事宜请示`，不得回退通用模板。
- 锡林浩特市临时用地是否涉及文物请示：用户说“锡林浩特”“锡林浩特市”“锡市”“胜利矿区”“胜利煤田”“胜利矿田”，且文种为文物保护许可申请、文物核查请示、查询用地是否涉及文物或临时用地涉文查询时，优先使用 `模板——锡林浩特市临时用地是否涉及文物请示`，不得回退通用模板；受文单位通常为“锡林浩特市文体旅游广电局”。如材料不是临时用地，保留锡林浩特模板的受文单位、标题口径和申请核查句，但删除或替换“临时用地”“临时车辆行驶道路”等不符合事实的表述。
- 申请开展考古勘探工作请示：用户说“勘探请示”“开展勘探请示”“申请勘探”，且未明确验收、成果备案、计划备案时，优先按本类处理。
- 考古勘探工作支持请示：用户说“协助勘探”“工作支持”“上报考古院支持勘探”，或项目涉及长城、金界壕等重点文物段落、资金压力、工期紧张、技术难度时，按独立文种处理。
- 勘探计划备案请示：默认附件写完整项目名称版“考古调查、勘探工作计划”；呼伦贝尔市项目还需勘探公司资质材料或备案资料。
- 勘探报告备案请示：附件写完整项目名称版“考古调查、勘探工作报告”或材料正式报告名。
- 勘探验收请示/函：默认使用新模板“关于申请XX项目考古勘探验收的函”，除非用户或材料明确要求旧模板。
- 考古勘探项目验收/检查意见书：作为意见书本体生成时，鄂尔多斯项目标题写“考古勘探项目检查意见书”；非鄂尔多斯项目按材料或用户指定，可继续使用“考古勘探项目验收意见书”。
- 伊金霍洛旗验收及开工：市级及以上项目使用“伊旗模版——xxx公司关于xxx项目选址范围内文物勘探工作验收及开工的请示”；旗级项目才使用只申请开工、不同时请示验收的开工请示口径。附件按长庆油田参考件完整口径先列全，第一项为“立项核准文件”，再列文物审查意见函、坐标、KML、宗地图、考古调查勘探报告、营业执照和负责人或法定代表人身份证复印件，实际出件时可删减。
- 文物调查报告、市级调查报告、自治区调查报告：不属于本 skill 内置文种；转用 `gongwen-survey-report`。

## 资料需求

先从用户材料中识别信息，不因普通缺项拖住第一版 Word。只有文种、办理阶段、发文主体或收文关系无法判断且会明显错向时才集中追问。

通常需要识别：

- 项目名称
- 发文主体或建设单位
- 办理阶段或目标文种
- 项目地点
- 项目面积或项目用地面积
- 建设规模及主要建设内容
- 可判断报送对象的模板、复函、审批意见或用户说明

联系人和电话没有材料时不写。缺少资料时在交付说明中列明影响，不在正文留空项。
