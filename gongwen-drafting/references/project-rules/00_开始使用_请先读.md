# 开始使用

本资料库用于中文公文材料的整理、检索、改写和生成，重点服务文物手续类公文。使用前先按本页顺序读取入口文件，不要直接从零散材料或聊天记忆开始判断。

## 先读顺序

1. `AGENTS.md`
   - 最高优先级项目记忆入口。
   - 先确认写作规则、长期记忆边界、资料库结构纪律和各文种固定规则。
2. `0.资料索引/00-资料库规范化总览.md`
   - 查看当前资料库如何按 `T/P/K` 方法映射。
   - 本轮只建立规范和映射，不移动原始政策、模板、参考公文。
3. `agent_handoff/AI_AGENT_START_HERE.md`
   - 给 Codex、Trae Solo 或其他 AI 接手时使用。
   - 包含跨设备读取顺序、验证命令和禁止事项。
4. `0.资料索引/00-AI工作台.md`
   - 写公文时的 Obsidian / Codex 快速入口。
   - 从这里进入文种手册、模板内化、案例索引和成稿复盘。
5. `agent/README.md`
   - 本地 Agent 工具入口、索引、检索、生成和固定版式命令。

## 当前结构

本资料库按《智能工具结构文档》的方法采用 `T/P/K` 分层理解，但第一批规范化不大规模搬动原有目录。

| 层级 | 含义 | 当前对应位置 |
| --- | --- | --- |
| `T_工具区_程序模板与资料库` | 稳定工具能力、脚本、固定版式流程、可迁移 skill | `agent/`、`agent_handoff/skills/` |
| `K_工具资料库` | 稳定知识资产、规则、模板、政策、参考件、AI 编译索引 | `AGENTS.md`、`0.资料索引/`、`1.政策法规与规范/`、`2.公文模板/`、`3.公文参考/`、`agent_handoff/PROJECT_MEMORY_EXPORT.md` |
| `P_项目区_本次任务资料流转` | 单次任务的原始输入、过程稿、人工确认稿、最终成果 | 后续按任务新建，当前不迁移既有桌面成稿 |

## 常用命令

在资料库根目录运行：

```bash
python3 -m unittest agent/test_gongwen_agent.py
python3 agent/gongwen_agent.py index --ocr --ocr-max-pages 3 --coverage-per-type 5
python3 agent/gongwen_agent.py web --host 127.0.0.1 --port 8765
```

核查请示固定版式生成与校验：

```bash
python3 agent/make_hecha_docx.py agent/sample_hecha_project.json --out /tmp/核查请示.docx --audit
python3 agent/check_hecha_docx.py /tmp/核查请示.docx
```

## 修改纪律

- 每一次资料库结构、入口、规则、脚本或索引说明修改，都必须记录到 `0.资料索引/健康检查/2026-06-15-资料库规范化日志.md` 或后续同类日志。
- 修改日志应记录时间、对象、类型、原因、影响范围、验证方式、验证结果和待确认事项。
- 本轮不做 Web 页面改造，不把整个资料库做成页面。
- 第一批不移动 `1.政策法规与规范/`、`2.公文模板/`、`3.公文参考/`。
- 原始资料层保持原貌；AI 只读取、摘录、建索引、写说明和沉淀规则。

## 禁止事项

- 不依赖原电脑 `~/.codex/memories` 作为执行入口。
- 不把一次性用户要求写成长期硬规则。
- 不把 `.gongwen_agent/` 当作正式项目区或迁移内容。
- 不把临时输出、桌面成稿、缓存文件混入工具区或知识库。
- 真实移动目录前，必须先检查 Obsidian 链接、脚本相对路径和迁移包清单。
