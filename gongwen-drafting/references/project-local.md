# 待出公文项目本地规则

## 定位

本文件记录 `/Users/drevan01/Desktop/待出公文` 工作区的项目级公文协作规则。它优先约束本项目内的公文写作、修改和 skill 迭代；稳定后如需复用到其他工作区，再同步到全局 `gongwen-drafting` skill。

## 本项目默认流程

1. 先从当前项目文件夹和用户对话中识别项目名称、建设单位、办理阶段、材料类型、地区和拟出文种。
2. 能判断文种和发文关系时，直接按 `routing.md`、`drafting-rules.md`、`word-output.md` 生成或修改 Word。
3. 无法判断且继续生成会错向时，集中问最关键问题，通常只问文种/办理阶段、发文主体、受文单位之一。
4. 生成或修改后的 `.docx` 默认输出到桌面；项目目录内只保留原始项目材料、规则和 skill 资源。
5. 对用户指出的 Word 问题做外科式修正，避免修一个版式区时破坏其他已确认区域。
6. 生成带红头的请示、函或许可/核查类公文时，红头信息行必须包含正式文号和签发人；暂缺时固定保留 `【文号】`、`【签发人】` 文字占位。

## 规则沉淀口径

- 用户明确说“固定”“记住”“写入规则”“更新项目记忆”“完善 skill”时，才把本轮经验写成长期规则。
- 项目个案附件取舍、临时受文单位、一次性措辞偏好，默认只进入当前成稿说明，不上升为长期规则。
- 疑似可复用但未确认的经验，先写入 `references/knowledge-index/规则候选/`，并注明来源项目、适用范围和风险。
- 已确认稳定的文种规则，写入 `references/knowledge-index/文种手册/` 或 `references/drafting-rules.md`；确定性生成和检查逻辑写入 `scripts/gongwen-agent/`。

## 本地材料使用边界

- 当前项目文件夹中的合同、图纸、PDF、坐标、报告等材料只作为当前出文事实来源，不自动复制进 skill。
- 模板、范本、可复用参考公文或已确认长期规则，才进入 `assets/`、`references/reference-documents/` 或 `references/knowledge-index/`。
- 不把桌面成稿、缓存、OCR 中间结果、运行日志写进 skill 包。

## 项目协作提示

处理任务前先判断信息状态：

- 开放区：信息足够时直接处理。
- 隐藏区：材料背景不足时只问关键缺项。
- 盲区：用户反复觉得不对时，主动指出可能错在文种、受文关系、模板路由或版式边界。
- 未知区：流程未定时给出多个可比较方案，再和用户一起收敛。
