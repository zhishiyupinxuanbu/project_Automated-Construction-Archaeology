# 参考公文每日更新检查清单

## 适用场景

- 每日首次维护资料库或接手本项目时，检查 `3.公文参考/` 是否有新增公文。
- 用户向 `3.公文参考/` 放入新文件后，先用本清单整理归档，再提炼可复用规则。

## 检查步骤

1. 查看 `3.公文参考/` 根目录及各文种目录下是否有新增、未归档或临时文件。
2. 按现有目录结构优先归入“文种—地区—文件角色”位置，例如企业申请文件、旗区给市局申请文件、参考文件等。
3. 无法稳定判断文种、地区或文件角色的，先保留在待确认位置，并在交付说明或健康检查记录中说明原因。
4. 归档后检查是否需要补充 `0.资料索引/文件卡片/`、`0.资料索引/01-按文种索引.md`、`0.资料索引/02-模板索引.md`、具体文种手册或模板内化文件。
5. 从新增参考件提炼规律时，分别记录文种、地区、发文主体、受文单位、标题公式、正文结构、附件设置、落款和版式特征。
6. 只有经多份同类参考件、明确地区模板或用户确认支撑的规律，才更新为通用模板、地区模板、版式硬规则或脚本检查项。
7. 单份案例中看似特别的措辞、附件、段落或格式，先进入 `0.资料索引/规则候选/` 或健康检查记录，不直接覆盖硬规则。

## 归档原则

- 保留原始参考公文内容，不直接改写原文。
- 文件移动或命名应服务于检索，不为整齐而改变事实属性。
- 新增地区目录可以按实际材料创建，但不得把未知地区强行归入相近地区。
- 模板校正必须区分“通用模板”“地区模板”“单项目个案”。
- 版式校正重点看页边距、标题位置、正文字号、正文缩进、附件悬挂缩进、落款署名和日期对齐。

## 输出要求

- 需要对用户说明本次新增参考件归档到了哪些目录。
- 若发现可优化模板或版式的稳定证据，应说明拟更新的模板、手册、脚本或检查清单。
- 若证据不足，应只记录为候选规则，不写入 `AGENTS.md` 硬规则。
