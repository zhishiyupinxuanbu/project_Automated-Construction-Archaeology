# 固定版式生成与校验说明

本说明用于跨电脑稳定生成“文物保护许可申请/核查请示”Word 文档，并说明 skill 内置专项生成脚本中的红头、标题、正文、附件说明和落款固定版式参数。不要只依赖自然语言规则；优先使用这里的脚本、固定版式 Word 样张和 `scripts/gongwen-agent/gongwen_format.py` 中的等效段落参数。

给已经写好的 `.docx` 公文做格式调整时，按当前文种手册、地区模板和本说明中的固定版式参数做外科式局部修正；不再使用 `docformat-gui` 或通用一键格式化工具覆盖成稿。

最终调整版式时，必须以 `assets/fixed-format/固定版式——文物保护许可核查请示格式范本.docx` 作为固定版式 Word 样张，逐项核对红头、标题、正文、附件、落款的字体字号、段落缩进、附件悬挂缩进、制表位和落款对齐。

## 固定内容

- 红头：所有 Word 公文默认添加红色发文机关标志、文号/签发人行和红色分隔线；发文机关标志一般为“发文主体+文件”，字号按单位名称长度自适应，避免“文件”二字孤行。
- 红线：必须使用连续线条或 Word 段落边框实现，线条应有足够粗细，不得用破折号、长横线字符或连续下划线拼接。
- 文号/签发人：固定保留；材料或用户明确提供时填入具体文号和签发人，未提供时先保留“文号”“签发人：”栏目并提示确认。
- 标题：编排于红色分隔线下方，2 号小标宋，居中；回行应保持词意完整、排列对称，不随意断词。
- 正文：3 号仿宋，首行缩进按两个中文字符宽度计算；16pt 正文字号时为 32pt；不得用手打空格代替段落缩进。
- 附件：第一项连续起排为“附件：1. XXXXX”；第 2 项起序号与第 1 项“1.”对齐；附件内容和续行起字对齐；必须使用悬挂缩进或制表位，不得用连续空格撑出版式。
- 落款：成文日期右空四字；署名先基于日期居中，只有居中后换行或明显不稳时才单独靠右。
- 专项生成要求：使用资料库脚本生成或大幅重建勘探验收、开工请示、备案请示等 Word 公文时，可复用上述固定参数，不得用普通首行缩进、连续空格、手工空格或粗略右对齐临时凑版式。既有成稿需要调整时，同样按问题区域局部处理，不得使用通用一键格式化入口重排整篇。
- 核查请示默认附件完整保留：前置审批/核准文件、营业执照、法人身份证、坐标 Excel、KML、宗地图；即使材料未一次性提供齐全，也先写入附件栏，缺项只在交付说明中提醒补充或确认。
- 核查请示不默认加入考古调查勘探报告、文物保护方案、文物影响评估报告。
- 乌兰察布地区项目的文物核查请示优先使用当地模板口径：加红色发文单位抬头和红线，标题写“关于XX项目文物核查的请示”，附件按“项目备案告知书、企业法人营业执照、法定代表人身份证复印件、项目可研报告、空间范围及用地经纬度坐标 Excel 表、KML 格式坐标文件、用地宗地图”执行；此规则覆盖通用核查标题和附件默认项。
- 锡林浩特市临时用地是否涉及文物请示优先使用照片内化模板口径：受文单位写“锡林浩特市文体旅游广电局”，标题采用“关于申请查询项目用地是否涉及文物的请示”或“关于申请查询项目临时用地是否涉及文物的请示”口径，附件最小清单为“临时用地范围及电子版国家2000坐标”；照片遮挡处不得猜测。项目名称/材料出现“胜利矿区”“胜利煤田”“胜利矿田”且指向锡林浩特市时，也按该模板处理；如材料不是临时用地，替换不符合事实的临时用地表述，不回退通用模板。

## 生成命令

先复制并编辑项目要素：

```bash
cp scripts/gongwen-agent/sample_hecha_project.json /tmp/hecha_project.json
```

生成 Word：

```bash
python3 scripts/gongwen-agent/make_hecha_docx.py /tmp/hecha_project.json --out /tmp/核查请示.docx --audit
```

如不指定 `--out`，文件默认保存到电脑桌面；只有在无桌面环境时才退回 `.gongwen_agent/outputs/`。

## 校验命令

```bash
python3 scripts/gongwen-agent/check_hecha_docx.py /tmp/核查请示.docx --report /tmp/核查请示-版式校验.md
```

校验未通过时，不要手工大范围重排；只按报错项局部修改。

## 文件说明

- `scripts/gongwen-agent/gongwen_format.py`：固定版式核心函数，红头、标题、正文、附件、落款分区处理；其中附件和落款参数可跨文种复用。
- `scripts/gongwen-agent/make_hecha_docx.py`：从 JSON 生成核查请示 Word。
- `scripts/gongwen-agent/check_hecha_docx.py`：校验核查请示 Word 是否符合固定规则。
- `scripts/gongwen-agent/sample_hecha_project.json`：项目要素示例。
- `assets/fixed-format/固定版式——文物保护许可核查请示格式范本.docx`：固定版式格式范本，内容使用“项目A、xx市、附件1、附件2”等中性占位，重点展示红头、红线、页边距、字体字号、缩进、附件对齐和落款位置。

## 修改纪律

后续修改 Word 公文时，只改用户指出的问题。标题、正文、附件、落款是独立区域，修某一区域不得连带修改其他区域；确需连带调整，必须先说明原因并确认。
