# 时间校验硬闸门

## 适用文件

涉及以下文件或文种时，生成前必须执行时间校验：

- 文物调查报告
- 文物复函/回函
- 勘探计划
- 勘探请示，含申请开展考古勘探工作请示、勘探计划备案请示
- 勘探报告

## 硬规则

1. 勘探请示必须发生在文物调查工作结束之后。
2. 勘探请示日期不得早于企业取得文物复函/回函日期；同一天可以，取得回函后 2-3 天再报也可以。
3. 勘探计划所写勘探开始日期必须在勘探请示日期后 2-3 天。
4. 勘探计划或勘探报告中的勘探开始日期不得晚于结束日期。
5. 当实际探工进场时间与计划或报告中写的勘探时间不完全一致时，以计划/报告正文所写勘探时间作为图片水印校验范围；脚本必须尝试检测计划和报告插入图片的水印日期。识别到水印日期的图片必须落入该时间范围。
6. 任一时间缺失、冲突、顺序不合规，或已识别到的图片水印日期超出范围，必须立刻停止生成并把错误返回给用户，不得自动改日期、删图片或继续生成；未识别到水印日期的图片不作为硬错误，但必须提示用户“未识别到水印日期，建议人工核对照片时间”。

## 工具入口

生成前由工作流自动调用：

```bash
python scripts/validate_project_timeline.py --form /path/to/智能报告信息填报表.xlsx --project-dir /path/to/project-source --photos-dir /path/to/photos
```

检查已经生成或已有的计划/报告 Word 内嵌图片时，传入 `--docx`；脚本会抽取 `word/media/` 里的图片，并默认调用本地 Paddle OCR 尝试识别水印日期。识别到水印日期的图片参与范围校验，未识别到水印日期的图片会汇总提示用户人工核对：

```bash
python scripts/validate_project_timeline.py --form /path/to/智能报告信息填报表.xlsx --docx /path/to/计划或报告.docx
```

若图片文件名、同名 `.txt`、同名 `.ocr.txt` 或 OCR JSON 缓存中无法读取水印日期，但需要现场识别，可显式加：

```bash
--ocr-images
```

OCR 默认使用本地 Paddle OCR：

```text
/Users/drevan01/Desktop/OCR
```

## 字段来源

优先使用智能填报表中的 `开始日期`、`结束日期`、`勘探时间`、`请示日期`、`回函日期`、`复函日期`、`文物调查结束日期` 等字段。

若传入项目源资料目录，脚本会补充读取文件名和可解析文本中的日期，重点识别文物调查报告、文物复函/回函、勘探请示、勘探计划和勘探报告。
