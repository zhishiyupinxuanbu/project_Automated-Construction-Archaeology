# 智能模板字段计算规则（批次1）

适用文件：

`智能生成报告技能资料/知识库/3.模板与表单/模板/报告、计划/北京卓凡文博技术有限公司-基准模板/人员1/智能报告生成基准模板_北京卓凡文博技术有限公司_无遗迹.docx`

## 1. 项目区域概况

### 地理位置、气候条件、历史沿革

模板字段：

- `{{项目名称}}坐落于{{项目位置}}，该项目面积为{{项目面积}}。`
- `{{项目所在地旗县地理位置概况}}`
- `{{项目所在地旗县行政区划与社会经济概况}}`
- `{{项目所在地旗县气候条件}}`
- `{{项目所在地旗县历史沿革}}`

生成规则：

- 从 `{{项目位置}}` 中识别旗县级行政区，例如“伊金霍洛旗”。
- 生成报告时调用联网/API检索旗县概况，优先使用政府官网、地方志、统计公报、行政区划资料等可靠来源。
- 输出为正式报告正文，不保留网页链接、百科超链接和来源痕迹。
- 历史沿革应按时间顺序写，避免散文化评价。
- “地理位置”开头固定先写项目本体信息，即：`{{项目名称}}坐落于{{项目位置}}，该项目面积为{{项目面积}}。`，后续再介绍所在旗县情况。

## 2. 文物概况

模板字段：

- `{{IF_有文物审查意见}}...{{ENDIF}}`
- `{{文物审查意见文件名}}`
- `{{文物审查意见文号}}`
- `{{文物审查意见结论}}`
- `{{IF_无回函_未发现文物}}...{{ENDIF}}`
- `{{IF_无回函_涉及文物}}...{{ENDIF}}`
- `{{涉及文物名称及情况}}`

生成规则：

1. 如果基础资料中有文物审查意见函，优先解析回函标题、文号和结论。
   - 输出格式：`根据《{{文物审查意见文件名}}》{{文物审查意见文号}}文件，{{文物审查意见结论}}。`
   - `{{文物审查意见结论}}` 只说明项目选址/用地范围是否涉及文物、古文化遗存、不可移动文物或保护范围；不得写入“原则同意建设”、施工中发现文物后的停工报告要求、办理手续要求、审批条件等多余行政意见。
   - 文件名中含 `文旅/文化旅游/文化和旅游/文物/复函/函`、且含“关于”或文号特征的 PDF，即使文件名初筛分数较低，也应读取正文或 OCR 文本后重新判断。
   - 扫描件回函优先使用本地 PaddleOCR 读取；不可用时再回退 RapidOCR、tesseract 等后备 OCR。
2. 如果没有回函，且调查未发现地表文物：
   - 输出：`根据本次考古调查工作，该项目用地范围地表并未涉及文物。`
3. 如果没有回函，但文物部门提供项目涉及文物信息：
   - 输出：`根据文物部门提供的信息，项目用地范围涉及{{涉及文物名称及情况}}。`

三种情况只能启用一种。

若项目资料中可识别到文物部门回函，但人工填写表或生成稿仍走 `无回函` 分支，专项检查必须提示 `回函信息待复核`，不得静默通过。

## 3. 勘探单元规格与探孔编号

模板字段：

- `{{勘探单元规格}}`，示例：`400米×400米`
- `{{普探列最大编号}}`
- `{{普探行最大编号}}`
- `{{重点勘探列最大编号}}`
- `{{重点勘探行最大编号}}`

计算规则：

- 设勘探单元边长为 `L` 米。
- 普探孔距一般为 `2米×2米`，列/行最大编号按 `L / 2 * 2 - 1` 形成奇数编号尾号。
  - 400米单元：`399`
  - 200米单元：`199`
  - 100米单元：`99`
  - 50米单元：`49`
  - 500米单元：`499`
- 重点勘探孔距一般为 `1米×1米`，列/行最大编号按 `L * 2 - 1` 形成奇数编号尾号。
  - 400米单元：`799`
  - 200米单元：`399`
  - 100米单元：`199`
  - 50米单元：`99`
  - 500米单元：`999`

## 4. 地面踏查人员

模板字段：

- `{{踏查技师数量}}`
- `{{踏查探工数量}}`

生成规则：

- 从“勘探队伍人员构成”或人员信息表统计。
- 若一个角色列出多名人员，以顿号、逗号、空格等分隔后计数。
- 若探工为“某某等26人”，优先识别数字 `26`。

## 5. 准确性控制

- 所有自动生成字段均应进入检查 XLSX；内部 `生成检查报告.txt` 只作为过程输入，默认转存到 `过程资料/生成检查/项目名称/`，不留在最终报告文件夹。
- 回函、人员数量、勘探单元规格属于高风险字段，缺失时不得静默生成，应提示人工确认。
- `基础信息/` 只读，不写入。

## 6. 勘探队伍人员构成

模板字段：

- `{{探工数量}}`
- `{{进场人数}}`
- `{{每日在场人员下限}}`
- `{{探工人员描述}}`
- `{{技师数量}}`

计算规则：

- `{{探工数量}}` 从“探工”一行提取，优先识别“等26人”这类数字。
- `{{进场人数}}` 必填；为空时必须提醒人工填写，不再由 `{{探工数量}}` 反推。`{{探工数量}} = {{进场人数}} - 8`。
- `{{每日在场人员下限}} = {{进场人数}} - 5`。
- “探工”表格行统一写作：`{{探工人员描述}}等{{探工数量}}人`。
- `{{技师数量}}` 从技师名单中按顿号、逗号、空格等分隔计数。

## 7. 勘探成果结构

“四、勘探成果”固定为三个小节，不再增加其他标题：

- `（一）地层堆积`
- `（二）标准孔`
- `（三）遗迹`

### 剖线地层堆积

模板块：

- `{{#剖线地层堆积}}`
- `{{图:剖线图}}`
- `剖线{{剖线编号}}地层堆积情况：`
- `{{剖线地层描述}}`
- `{{剖线地层描述}}` 填入完成后必须追加分页符，下一张剖面图或下一条剖线说明另起页。
- 剖线/剖面图、图题和地层堆积说明所在页必须保留基准模板横向分节符；展开重复块或清理空占位时不得删除横向 `sectPr`，剖面图组结束后必须恢复纵向分节。
- `{{/剖线地层堆积}}`

生成规则：

- 用户在输入表中人工填写每条剖线的地层描述。
- 生成器按原报告格式输出：先图题，再剖线编号，再分层描述。
- 剖线数量由输入表统计后写入 `{{剖线数量}}`。

### 标准孔

模板块：

- `{{#标准孔详情}}`
- `（{{序号}}）勘探单元{{勘探单元}}`
- `{{标准孔编号}}`
- `X：{{X坐标}}，Y：{{Y坐标}}，高程：{{高程}}`
- `{{标准孔地层描述}}`
- `{{图:标准孔位置图:勘探编号}}`
- `{{图:标准孔土样照:勘探编号}}`
- `{{/标准孔详情}}`

生成规则：

- 每个标准孔由输入表生成一个重复块。
- 标准孔数量由输入表统计后写入 `{{标准孔数量}}`。
- 标准孔详情表以 `标准孔坐标.xlsx` 为主键：`勘探编号` 匹配位置图，`探孔编号` 匹配地层描述和土样照，不能按图片队列顺序推断。

### 剖面探孔记录表

生成规则：

- `考古勘探剖面探孔记录表` 按 `剖线坐标.xlsx` 生成，剖线 A/B 分别绑定 `A-A′`、`B-B′` 剖面图。
- 每个探孔的层厚必须来自对应剖面图 PDF 或可抽取文本中的逐孔标注，不得使用固定数组、相邻探孔推测值、整条剖线概括描述或历史项目数据。
- 剖面图 PDF 文本中的层厚数字应按图面横向位置归属到最近的探孔编号，再按上下顺序作为第①层、第②层等层厚；不得把所有 `0.xxm` 数字按出现顺序铺平后两两分组。
- 若无法可靠抽取逐孔层厚，应在报告对应表格中写入 `【插入错误：未能从剖面图可靠提取逐孔层厚】`，并同步进入专项检查表和问题摘要，不得静默生成看似完整但与图不一致的表格。

## 8. 结论综合

模板字段：

- `{{勘探成果综合结论}}`

生成规则：

- 生成器根据剖线地层描述、标准孔地层描述、遗迹情况综合生成。
- 无遗迹模板中应明确“未发现古代文化遗存”，并概括主要地层类型、土色土质、包含物、是否为自然沉积。
- 若项目现场记录中已经有明确 `结论：...`，可直接作为 `勘探成果综合结论` 调用并插入；有分区项目缺少逐区地层总结时，也可优先使用该现场记录结论，不再标记缺少分区地层逐区总结。
- 不直接沿用固定旧项目结论。

## 9. 人工填报表与自动识别

填报表：

- `过程资料/智能报告信息填报表.xlsx`

工作表：

- `项目基础信息`：项目名称、建设单位、勘探单位、项目位置、项目面积、调查面积、勘探面积、勘探时间、报告年月、遗迹结论等。
- `项目区域概况`：项目所在地旗县及地理位置、行政区划、气候条件、历史沿革等正文。
- `文物概况`：三选一类型及回函标题、文号、结论或涉及文物情况。
- `勘探参数`：勘探单元规格、孔距、探孔编号尾号等。
- `现场限制`：是否存在不可勘探区域、不可勘探原因。
- `人员构成`：技师、探工、现场人员等；未填写派生数量时由生成器自动计算。
- `自动生成字段`：报告年月、剖线数量、标准孔数量、勘探成果综合结论等；默认不用填写，必要时可人工覆盖。
- `剖线地层堆积`、`标准孔`：用于生成“四、勘探成果”中的重复内容。
- `图片清单`：暂只保留基础表头；图片位规则未最终确定前，不预置具体图片位。

生成脚本：

- `智能生成报告技能资料/工具脚本/fill_smart_template_from_form.py`

默认输入：

- 表格：`过程资料/智能报告信息填报表.xlsx`
- 模板：`智能生成报告技能资料/知识库/3.模板与表单/模板/报告、计划/北京卓凡文博技术有限公司-基准模板/人员1/智能报告生成基准模板_北京卓凡文博技术有限公司_无遗迹.docx`

默认输出：

- `生成报告/{{项目名称}}-智能生成勘探报告.docx`
- `生成报告/{{项目名称}}-专项检查表.xlsx`

执行方式：

```bash
python3 智能生成报告技能资料/工具脚本/fill_smart_template_from_form.py
```

报告年月规则：

- 优先由 `结束日期` 推导，例如 `2025年11月28日` 自动生成 `2025年11月`。
- 如果 `结束日期` 为空，生成器会尝试从 `勘探时间` 中识别月份。
- 如需强制指定，可在 `自动生成字段` 工作表填写 `报告年月`。
